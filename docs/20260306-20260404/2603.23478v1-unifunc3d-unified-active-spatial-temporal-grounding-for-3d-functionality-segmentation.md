## 摘要
三维场景中的功能性分割要求智能体将隐式的自然语言指令定位到细粒度交互元素的精确掩码上。现有方法依赖于碎片化的流水线，在初始任务解析阶段存在视觉盲区。我们观察到，这些方法受限于单尺度、被动且启发式的帧选择。我们提出了 UniFunc3D，这是一个统一且无需训练的框架，它将多模态大语言模型视为主动观察者。通过将语义、时间及空间推理整合到单次前向传播中，UniFunc3D 执行联合推理，从而将任务分解建立在直接的视觉证据之上。我们的方法引入了采用由粗到精策略的主动时空定位。这使得模型能够自适应地选择正确的视频帧，并专注于高细节的交互部分，同时保留消除歧义所需的全局上下文。在 SceneFun3D 数据集上，UniFunc3D 达到了最先进的性能，在没有任何特定任务训练的情况下，以 59.9% 的 mIoU 相对提升大幅超越了无需训练和基于训练的方法。代码将在我们的项目页面发布：https://jiaying.link/unifunc3d。

## Abstract
Functionality segmentation in 3D scenes requires an agent to ground implicit natural-language instructions into precise masks of fine-grained interactive elements. Existing methods rely on fragmented pipelines that suffer from visual blindness during initial task parsing. We observe that these methods are limited by single-scale, passive and heuristic frame selection. We present UniFunc3D, a unified and training-free framework that treats the multimodal large language model as an active observer. By consolidating semantic, temporal, and spatial reasoning into a single forward pass, UniFunc3D performs joint reasoning to ground task decomposition in direct visual evidence. Our approach introduces active spatial-temporal grounding with a coarse-to-fine strategy. This allows the model to select correct video frames adaptively and focus on high-detail interactive parts while preserving the global context necessary for disambiguation. On SceneFun3D, UniFunc3D achieves state-of-the-art performance, surpassing both training-free and training-based methods by a large margin with a relative 59.9\% mIoU improvement, without any task-specific training. Code will be released on our project page: https://jiaying.link/unifunc3d.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：3D场景中的“功能性分割”（Functionality Segmentation）要求智能体根据隐式指令（如“打开顶灯”）定位并分割出细粒度的交互部件（如墙上的开关）。
*   **研究动机**：现有方法（如Fun3DU）存在三大缺陷：
    1.  **视觉盲区**：初始任务解析依赖纯文本LLM，未见场景即进行分解，易导致语义误判。
    2.  **被动启发式选择**：依赖手工设计的规则选择视频帧，缺乏时空一致性，难以处理复杂空间关系。
    3.  **单尺度限制**：缺乏类似人类的“缩放”机制，导致微小功能部件（如旋钮、开关）在低分辨率下变成不可见的噪声。
*   **整体含义**：UniFunc3D 旨在通过一个统一的、无需训练的多模态大模型（MLLM）框架，将感知与推理缝合，消除碎片化流程带来的误差累积。

### 2. 方法论
UniFunc3D 将 MLLM 视为“主动观察者”，通过单次前向传播整合语义、时间和空间推理。
*   **核心思想**：采用**由粗到精（Coarse-to-Fine）的主动时空定位**策略。
*   **关键技术流程**：
    1.  **第一阶段：主动时空定位与目标识别**
        *   **粗略轮（Round 1）**：多轮采样（Multi-sampling）。在低分辨率下对全视频进行多次偏移采样，让模型识别功能物体名称、选择最佳候选帧并给出初始定位点。
        *   **精细轮（Round 2）**：原生分辨率缩放。在候选帧周围提取密集时间窗口，以原生高分辨率处理，在保留全局上下文的同时实现“视觉缩放”，修正初始定位。
    2.  **第二阶段：视觉掩码生成与验证**
        *   使用 **SAM3** 根据定位点生成候选掩码。
        *   **视觉验证**：将掩码以红色叠加层（Overlay）形式展示给 MLLM，由模型判断该区域是否仅包含目标部件，过滤过度分割的错误。
    3.  **3D 提升与共识**：利用相机位姿将 2D 掩码投影至 3D 点云，通过多视角一致性（Multi-view Agreement）投票生成最终 3D 掩码。

### 3. 实验设计
*   **数据集**：**SceneFun3D**（目前唯一的 3D 场景功能性分割数据集），包含 230 个高分辨率室内场景扫描和 3000 多个挑战性任务描述。
*   **Benchmark**：分为 split0（30个验证场景）和 split1（200个训练场景）。
*   **对比方法**：
    *   **开放词汇 3D 分割**：OpenMask3D, LERF, OpenIns3D。
    *   **基于训练的方法**：TASA (72B), AffordBot (72B)。
    *   **无需训练的方法**：Fun3DU (9B)。
*   **评估指标**：AP25, AP50, AR25, AR50, mIoU。

### 4. 资源与算力
*   **硬件环境**：使用了 **8 张 NVIDIA H100 GPU**。
*   **推理效率**：
    *   UniFunc3D 处理每个场景平均耗时约 **26 分钟**。
    *   相比基准方法 Fun3DU（约 82 分钟/场景），实现了 **3.2 倍的加速**。
*   **模型规模**：使用了 Qwen3-VL 的 8B 和 30B 版本。

### 5. 实验数量与充分性
*   **实验规模**：在全部 230 个场景上进行了测试，涵盖了 split0 和 split1 两个子集。
*   **消融实验**：非常充分，分别验证了：
    *   单阶段 vs 两阶段架构的差异。
    *   多轮采样（K 次迭代）对性能的影响（K=1 到 K=8）。
    *   时间窗口（Temporal Window）和视觉验证（Verification）的增益。
*   **客观性**：对比了参数量更大的 72B 模型以及经过 1000 轮微调的训练型模型，证明了其无需训练（Training-free）架构的优越性。

### 6. 主要结论与发现
*   **性能领先**：UniFunc3D-30B 在所有指标上均达到 SOTA。在 mIoU 上比 Fun3DU 提升了 **59.9%**。
*   **超越训练模型**：尽管无需针对任务进行微调，UniFunc3D 依然显著超越了需要大量 3D 标注数据训练的 AffordBot-72B。
*   **时空推理的重要性**：实验证明，密集的时间窗口和多轮采样是解决空间歧义（如“左侧第二个抽屉”）的关键。

### 7. 优点
*   **统一性**：消除了“视觉盲目”的文本推理阶段，减少了级联误差。
*   **主动性**：通过主动采样替代了死板的手工启发式规则。
*   **人类化感知**：由粗到精的策略在不丢失全局信息的前提下解决了微小目标的识别难题。
*   **高效且免训练**：在提升精度的同时大幅缩短了推理时间，且具有极强的泛化能力。

### 8. 不足与局限
*   **极小目标挑战**：对于占图像面积小于 0.1% 的极端微小部件，定位仍具挑战。
*   **遮挡问题**：在严重遮挡的场景下，仅靠视频帧采样可能无法完全还原几何关系。
*   **交互缺失**：目前仍是静态离线处理，尚未支持机器人在线交互式修正。
