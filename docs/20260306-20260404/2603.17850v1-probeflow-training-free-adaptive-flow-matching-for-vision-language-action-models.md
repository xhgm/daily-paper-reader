## 摘要
最近，配备流匹配（Flow Matching, FM）动作头的视觉-语言-动作（VLA）模型在复杂的机器人操控任务中取得了最先进的性能。然而，FM 所需的多步迭代常微分方程（ODE）求解引入了推理延迟，阻碍了响应式的物理控制。虽然目前的加速工作主要优化视觉-语言模型（VLM）主干网络，但动作头的瓶颈仍被忽视。为解决这一问题，我们提出了 ProbeFlow，这是一种专为连续机器人控制量身定制的无需训练的自适应推理框架。通过利用初始速度向量与前瞻速度向量之间的余弦相似度来评估几何轨迹复杂度，ProbeFlow 能够动态调度积分步数，从而剪枝冗余的网络评估。在 MetaWorld 基准测试中，它将动作解码速度提升了 14.8 倍（平均步数从 N = 50 减少到 2.6），并在不降低操控成功率的情况下将端到端系统延迟降低了 2.8 倍。在长时程 LIBERO 基准测试中，探针会自动分配更密集的调度以应对语义瓶颈，从而有效解决了流求解器的延迟问题。真实世界的物理部署证实，ProbeFlow 在确保执行稳定性的同时，成功缓解了动作解码延迟，为低延迟连续生成式策略提供了一种高度实用的解决方案。

## Abstract
Recent Vision-Language-Action (VLA) models equipped with Flow Matching (FM) action heads achieve state-of-the-art performance in complex robot manipulation. However, the multi-step iterative ODE solving required by FM introduces inference latency that precludes responsive physical control. While current acceleration efforts optimize the Vision-Language Model (VLM) backbone, the action head bottleneck remains overlooked. To address this, we propose ProbeFlow, a training-free adaptive inference framework tai- lored for continuous robotic control. By evaluating geometric trajectory complexity via the cosine similarity between initial and lookahead velocity vectors, ProbeFlow dynamically sched- ules integration steps to prune redundant network evaluations. On the MetaWorld benchmark, it accelerates action decoding by 14.8x (reducing average steps from N = 50 to 2.6) and cuts end-to-end system latency by 2.8x without compromising the manipulation success rate. On the long-horizon LIBERO benchmark, the probe automatically allocates a denser schedule to navigate semantic bottlenecks, effectively resolving the flow solver delay. Real-world physical deployments confirm that ProbeFlow successfully mitigates action decoding latency while ensuring execution stability, offering a highly practical solution for low-latency continuous generative policies.

*   **核心问题**：视觉-语言-动作（VLA）模型在机器人控制中表现优异，但其采用的**流匹配（Flow Matching, FM）**动作头存在严重的推理延迟。FM 需要通过多步迭代求解常微分方程（ODE）来生成动作序列，这导致在实时控制中，动作解码成为了整个系统的计算瓶颈（通常占总延迟的 70% 以上）。
*   **研究背景**：现有的加速方案多集中在压缩视觉-语言模型（VLM）的主干网络，而忽略了动作头。作者观察到，FM 生成的概率路径在不同阶段复杂度不同：简单移动时路径趋于直线，而精细操作时路径弯曲。因此，使用固定的高频采样步数会造成巨大的计算浪费。

### 2. 论文提出的方法论：ProbeFlow
*   **核心思想**：提出一种**无需训练（Training-free）**的自适应推理框架，通过几何手段评估轨迹复杂度，动态调整 ODE 求解步数。
*   **关键技术细节**：
    *   **前瞻线性探针（Lookahead Linearity Probe）**：在 $t=0$ 时刻计算初始速度 $v_{start}$，并向前推进一个较大的步长（如 $\Delta t=0.5$）得到预测状态 $x_{probe}$，随后计算该点的速度 $v_{probe}$。
    *   **余弦相似度评估**：计算 $v_{start}$ 与 $v_{probe}$ 之间的余弦相似度 $S$。若 $S \approx 1$，说明轨迹在该段非常平直；若 $S \ll 1$，说明轨迹存在显著曲率。
    *   **动态步数调度**：根据 $S$ 值将步数 $N$ 映射到 $[N_{min}, N_{max}]$ 区间。在线性区域，算法会复用探针阶段计算出的速度，直接跳过中间步骤，实现极低开销的推理。
*   **算法流程**：单次推理中，先执行一次“前瞻”探测，根据探测到的几何线性度决定后续是采用稀疏积分（复用探测结果）还是密集积分。

### 5. 实验数量与充分性
*   **实验规模**：在 MetaWorld 的 50 个任务和 LIBERO 的多个长程任务上进行了广泛测试，每个任务运行 10 个回合并跨越 5 个随机种子，提供了标准差分析。
*   **消融实验**：针对前瞻步长 $\Delta t_{probe}$、线性阈值 $\epsilon$ 进行了详细的参数敏感性分析，验证了“前瞻半步（0.5）”是性能与效率的最佳平衡点。

### 6. 主要结论与发现
*   **大幅加速**：在 MetaWorld 上，动作头解码速度提升了 **14.8 倍**（平均步数从 50 降至 2.6），端到端系统延迟降低了 **2.8 倍**。

*   **自适应性**：可视化证明 ProbeFlow 能在物体抓取等关键瓶颈处自动增加步数，而在空载移动阶段自动减少步数。

*   **即插即用**：无需重新训练模型或微调参数，具有极高的实用价值。
