## 摘要
文本到点云 (T2P) 定位旨在根据自然语言描述推断 3D 点云地图中的精确空间位置，这反映了人类如何通过语言感知和交流空间布局。然而，现有方法在很大程度上依赖于浅层的文本-点云对应关系，缺乏有效的空间推理，限制了其在复杂环境中的准确性。为了解决这一局限性，我们提出了 VLM-Loc，这是一个利用大型视觉语言模型 (VLM) 的空间推理能力进行 T2P 定位的框架。具体而言，我们将点云转换为鸟瞰图 (BEV) 图像和场景图，共同编码几何和语义上下文，为 VLM 提供结构化输入，以学习连接语言和空间语义的跨模态表示。在这些表示的基础上，我们引入了一种部分节点分配机制，将文本线索与场景图节点显式关联，从而实现可解释的空间推理以进行精确地定位。为了促进跨不同场景的系统评估，我们提出了 CityLoc，这是一个基于多源点云构建的细粒度 T2P 定位基准。在 CityLoc 上的实验表明，与现有最先进的方法相比，VLM-Loc 实现了卓越的准确性和鲁棒性。我们的代码、模型和数据集可在 \href{https://github.com/MCG-NKU/nku-3d-vision}{代码库} 获取。

## Abstract
Text-to-point-cloud (T2P) localization aims to infer precise spatial positions within 3D point cloud maps from natural language descriptions, reflecting how humans perceive and communicate spatial layouts through language. However, existing methods largely rely on shallow text-point cloud correspondence without effective spatial reasoning, limiting their accuracy in complex environments. To address this limitation, we propose VLM-Loc, a framework that leverages the spatial reasoning capability of large vision-language models (VLMs) for T2P localization. Specifically, we transform point clouds into bird's-eye-view (BEV) images and scene graphs that jointly encode geometric and semantic context, providing structured inputs for the VLM to learn cross-modal representations bridging linguistic and spatial semantics. On top of these representations, we introduce a partial node assignment mechanism that explicitly associates textual cues with scene graph nodes, enabling interpretable spatial reasoning for accurate localization. To facilitate systematic evaluation across diverse scenes, we present CityLoc, a benchmark built from multi-source point clouds for fine-grained T2P localization. Experiments on CityLoc demonstrate VLM-Loc achieves superior accuracy and robustness compared to state-of-the-art methods. Our code, model, and dataset are available at \href{https://github.com/MCG-NKU/nku-3d-vision}{repository}.

    *   **人类逻辑模拟**：人类通过感知周围物体及其相对方位来确定位置。大语言视觉模型（VLM）具备强大的多模态推理能力，能够理解复杂的空间描述并将其与视觉输入对齐，这为解决细粒度定位提供了新思路。

### 2. 方法论：核心思想与关键技术
VLM-Loc 框架的核心思想是将 3D 空间感知转化为 VLM 擅长的 2D 视觉推理任务。
*   **多模态地图表示**：
    *   **BEV 图像渲染**：将 3D 点云投影到水平面，生成带有颜色和语义信息的鸟瞰图（BEV），利用 VLM 在 2D 图像上的预训练知识。
    *   **场景图生成**：构建包含物体节点（语义标签、像素中心坐标）的场景图，为 VLM 提供结构化的语义和几何上下文。
*   **部分节点分配（Partial Node Assignment, PNA）**：
    *   **核心逻辑**：针对文本描述中某些物体可能在当前地图范围外或不可见的问题，PNA 机制显式地监督模型判断文本线索是否可被匹配到场景图节点。
    *   **判定准则**：通过计算物体可见区域中心与查询位置可见区域中心的距离，若小于阈值 $\tau$ 则视为可匹配。
*   **自回归位置估计**：
    *   将系统提示词、场景图、文本查询和 BEV 图像输入 VLM。
    *   模型以自回归方式输出 JSON 格式的字符串，包含节点匹配结果和最终的 2D 像素坐标 $(x, y)$。

### 6. 主要结论与发现
*   **性能飞跃**：VLM-Loc 在 CityLoc-K 测试集上的 Recall@5m 达到 35.91%，比之前的 SOTA 方法 CMMLoc 提升了 **14.20%**。
*   **推理的重要性**：实验表明，准确的节点分配（PNA）与定位误差呈强负相关，即模型对空间关系的理解越准，定位越精。
*   **方向线索最关键**：在文本描述中，方向信息对定位的贡献远大于颜色信息。
*   **泛化能力强**：在未见过的 CityLoc-C 场景中，VLM-Loc 的表现依然显著优于传统方法，证明了 VLM 具备更强的跨场景理解力。

### 7. 优点
*   **架构创新**：首次系统性地利用大模型（VLM）的推理能力解决 T2P 定位，打破了传统特征匹配的瓶颈。
*   **可解释性**：通过 PNA 机制，模型不仅给出坐标，还给出了文本与地图物体的对应关系，使定位过程透明化。
*   **鲁棒性**：BEV + 场景图的组合有效弥补了单一表示的不足，对复杂城市环境适应性强。

### 8. 不足与局限
*   **推理延迟**：由于依赖大型 VLM，FPS 较低（<1 FPS），难以直接应用于高频实时定位系统。
*   **文本依赖**：目前使用模板生成的文本进行训练和测试，虽然高效，但与人类完全自由、非结构化的口语描述仍有一定差距。
*   **维度损失**：将 3D 点云压缩为 2D BEV 图像虽然简化了任务，但也可能丢失垂直方向（如楼层高度）的关键定位信息。
