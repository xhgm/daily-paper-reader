Abstract. This paper presents a new method for the zero-shot open-vocabulary semantic segmentation (OVSS) of 3D automotive lidar data. To circumvent the recognized image-text modality gap that is intrinsic to approaches based on Vision Language Models (VLMs) such as CLIP, our method relies instead on image generation from text, to create prototype images. Given a 3D network distilled from a 2D Vision Foundation Model (VFM), we then label a point cloud by matching 3D point features with 2D image features of these prototypes. Our method is state-of-the-art for OVSS on nuScenes and SemanticKITTI. Code, pre-trained models, and generated images are available at https://github.com/valeoai/ IGLOSS .

Keywords: open vocabulary · 3D segmentation · image generation 

# 1 Introduction 

We introduce a new framework for zero-shot open-vocabulary semantic segmen-tation (OVSS) of 3D automotive LiDAR data : given free text prompts defining classes of interest, our method labels scan points with one of these classes. Being 

zero-shot , thus not requiring any labels nor any sample, brings efficiency in a driving setting where manually labeling 3D data is long and costly [1]. Besides, while autonomous driving can simply rely on a closed set of classes, being open-vocabulary , not specific to fixed classes, brings flexibility. It effortlessly offers any segmentation specialization depending, e.g., on a target world region or on a spe-cific driving scenario. Such an OVSS method is not intended to be embedded in a vehicle but to be used, e.g., for auto-labeling or retrieval. OVSS has developed [16,17,116] with Vision Language Models (VLMs) built on the image modality [36, 80, 111] thanks to the availability of massive 2D data aligned with text, such as captioned images [8]. In contrast, there is much less 3D data available, even less for driving, and even less with associated text. It explains why, rather than directly address OVSS in 3D, most 3D methods leverage existing 2D VLMs [80] and transfer 2D knowledge to 3D via 2D-3D (pixel-point) correspondences, using datasets where both 2D and 3D modalities are acquired jointly. To obtain a pure 3D model, not requiring image input, the image encoder of the 2D VLM is distilled into a 3D backbone to align point features and pixel features [74], which are already aligned with text in the VLM.

While it is hard to escape image intermediation and 2D-3D distillation, we propose to use a radically different approach to bridge text and images [42], as overviewed in Fig. 1. Given a text query, we first use a text-conditioned image generator (IG) such as ChatGPT [67] to produce images that we use as class prototypes. Second, using a 2D Vision Foundation Model (VFM) that excels at dense downstream tasks, such as DINOv2 [68], we produce pixel-level features for the images, which we average and use to fit a multinomial logistic regression (LR) at test time. Third, given the 3D VFM distilled from the same 2D VFM, we use the LR classifier to assign labels to points according to their 3D features, which are by design aligned with the 2D features. To construct such an alignment, we build ScaLR +, a 2D-3D distillation scheme improving over ScaLR [78]. The advantages of our method, named IGLOSS (acronimized paper title), are: 

1. No difficult image-text alignment. Our method does not rely on the classical image-text alignment [80] used by most other 3D OVSS methods. It thus does not suffer from the inherent and recognized modality gap between images IGLOSS: Image Generation for Lidar OVSS 3

and text, as shown by theoretical and empirical studies [40, 53, 79, 108]. This issue is only partly addressed by alternative pretraining architectures [12, 110], additional losses [29, 48] and specific training recipes [18, 48, 48, 51]. In IGLOSS, text and image are bridged with the versatility of image generation from text. Besides, the 2D-3D alignment operates between closely related visual modalities. 

2. No large dataset to curate, no extra supervision. Bridging text and image in a VLM requires a large amount of curated data, pairing images and cap-tions [80]. Alternatively, aligning them with less data requires supervision, e.g., with object boxes [57, 112] or segmentation masks [28, 45, 112, 121]. In contrast, our method relies on image generation from text, which arguably requires less image-caption pairs [15] and is less sensitive to noisy (meaningless) captions [19]. Besides, diffusion or flow matching training data can also include images without text, e.g., when training with classifier-free diffusion guidance [33]. 

3. Better image features. Early 2D VLMs were trained at image level and low resolution, which suits downstream tasks like image classification or retrieval [80]. For dense tasks like segmentation, additional apparatuses have to be used [4, 16, 17, 50, 63, 88, 116]. Nevertheless, imprecise segment boundaries led a number of 3D OVSS methods, including recent ones [10, 26, 52, 69, 114, 120], to integrate some form of denoising or use an additional FM for segmentation, such as SAM [45]. Meanwhile, image VFMs such as DINOv2 provide state-of-the-art (SOTA) dense encoders for segmentation tasks [68], pushing some to improve a VLM via a VFM teacher [91, 103] or equip a VFM with text alignment [39]. Although recent 2D VLMs such as EVA-02-CLIP [24] and SigLIP 2 [94] are stronger, 2D VFMs remain SOTA for downstream segmentation tasks [90]. 

4. Foundation models as black boxes. Existing 3D OVSS methods re-quire “opening the box” of foundation models (FMs), e.g., to extract dense CLIP features [16,116] or to obtain masks via a cross-attention map in Stable Diffusion (SD) [83]. In contrast, our method uses FMs in a black box manner: any text-to-image generator, any 2D VFM, and any 3D backbone distilled from the same 2D VFM can do. If any of these FMs improves, so will our method, effortlessly. 

5. Simplicity. Most 3D OVSS methods [10, 26, 38, 41, 52, 69, 85, 92, 98, 101, 114,120] are fairly complex, combining several other methods to extract masks or captions, adding multiple distillation losses for alignment and regularization, in-cluding modules to denoise labels or features, or requiring several training stages or recipes (details in appendix). In contrast, IGLOSS is the simple composition of three FMs, which already exist, although we also improve the 3D VFM. 

6. No need to input accumulated scans or images. Some of the top-performing 3D OVSS methods operate on a sequence of scans [11, 101, 114, 120], or get their best results by additionally using images or scan sequences at test time [11, 38, 74, 92, 101]. Our method does not need such strategies, that have been argued to have high storage and computational costs [98]. 

7. Vocabulary and semantic concept openness , for all methods, is bounded by their training sets. For 3D OVSS, the bottleneck is the 2D-3D corre-spondence dataset, which is in practice much smaller than 2D VLM training sets. Broad class names (e.g., manmade [5]) and negative definitions (e.g., other flat ) have however to be made explicit, as other OV methods do too.

10. SOTA. Thanks to a better text-pixel alignment, an enhanced 2D-3D distillation, and the use of logistic regression for label assignment, we reach state-of-the-art results in zero-shot 3D OVSS on both nuScenes [5] and Semantic-KITTI [1]. Our method also very easily sup-ports the mixing of several 2D-3D VFMs for an increased performance. 

# 2 Related Work 

The introduction largely reviews related work. We focus here on 3D OVSS meth-ods applying to driving data and OV methods using image generation (IG). 6 N. Samet et al. 

3D OVSS methods for driving data. Tab. 1 lists 3D OVSS methods ap-plying to automotive lidar data and outlines their main characteristics. (Details on each method are in the appendix.) As can be seen, IGLOSS differs signifi-cantly: it is the only method to (1) bridge image and text with IG (“ t ⇒ i ”), (2) use a foundation model aligning 2D and 3D (“ i ⇒ f ⇐ p ”), and (3) assign labels using logistic regression. It is also the only one, along with SAL [69], to offer simultaneously (4) full vocabulary and concept openness, while not using (5) images nor (6) scan sequences at test time. Our code will be publicly released. 

Uses of image generation in open-vocabulary methods. Among 3D OVSS methods, only SAS [52] uses IG, but for different purposes than IGLOSS. It uses Stable Diffusion [83] (SD) to score VLMs in order to ensemble them. A few uses of IG to produce OV prototypes can be found in 2D. To achieve zero-shot OV image classification, SuS-X [95] uses few-shot classification tech-niques [113], providing a support set made of images generated from SD or retrieved from a vision-language data bank [87]. Other classification methods [14, 49] open the SD network, choosing the text conditioning that best predicts the noise added to the input image. It is however not applicable to segmentation. Closer to IGLOSS is OVDiff [42], that addresses OVSS in 2D. It uses SD both to generate images and to segment them. From prototype images, OVDiff generates two sets of prototype features for each class, representing foreground and background. To that end, unsupervised instance masks from CutLER [100] are selected using SD cross-attention maps, which identify pixels attending more strongly to text. Pixel features of these segments (foreground) and of their com-plement (background) are then extracted using a combination of off-the-shelf feature extractors [7, 80, 83], and clustered with k-means to capture part-level prototypes. Using ChatGPT to categorize classes as “thing” or “stuff”, OVDiff also includes a CLIP-based image filtering to possibly reject unfit generated im-ages, and a prototype rejection mechanism to prevent the background prototype for one class to match the foreground prototype of others. In contrast, IGLOSS is much simpler. To address the class prototype seg-mentation issue, the image generator is just asked to picture the class with a white background. No additional methods and models is needed, no sliding win-dow, no part-level clustering (we just average patch features in each image), no explicit background-foreground or thing-stuff separation. Notably, while OVDiff generates 32 images per class, a couple of images are enough for IGLOSS. 

Test-time adaptation (TTA) for Open Vocabulary tasks has emerged in the literature as a prominent general strategy to bridge the gap between pretrained models and flexible downstream tasks. It includes the exploitation of side knowledge, e.g., dictionary-based contextual prompts [104] or external more or less heavy models [56,102,104] to adjust prompts or segments. It also includes light optimization stages for test-time prompt tuning (TPT) [25,59], feature [64] or proposal reweighting [2], learning on-the-fly a classifier for novel classes [75], or more involved test-time training (TTT) [119]. These works, recognized as OV, demonstrate that adapting at test time to the target sample (or even distribution [43]) is a increasingly common mechanism for IGLOSS: Image Generation for Lidar OVSS 7

enhancing the performance of open-vocabulary methods. It justifies our in-place replacement of the classical nearest-neighbor (NN) classification by a logistic regression (LR) fitted at test time, which is in any case very fast ( ∼ 70ms). 

# 3 Method 

Our method generates prototype images from text prompts and labels the point cloud by matching 3D features with 2D features of these prototypes (Fig. 1). We describe here the 2D-3D distillation and the two inference stages: image prototype generation and point-cloud segmentation. We also show an annotation-free extension for closed-set segmentation and an easy way to ensemble VFMs. 

3.1 2D-3D Visual Foundation Model alignment 

Our key ingredient is a pair ⟨M, M ′⟩ of a 2D VFM M and an aligned 3D VFM M ′

.Some are publicly available, like ⟨OpenSeg, OpenScene ⟩ [74, 112], as part of existing 3D OVSS methods. However, they are also aligned with CLIP text features, which adds a constraint on 2D features that we do not require. 

ScaLR. One of the most prominent alignment of a strong 2D VFM and a 3D network is ⟨DINOv2, ScaLR ⟩. While “ScaLR” is originally the name of 2D-3D distillation method [78], it is also understood as is a multi-domain ScaLR distilla-tion of DINOv2 [68] into WaffleIron [77], jointly trained on several lidar datasets: nuScenes [5], SemanticKITTI [1], Panda64 and PandaGT [105]. As a pretrained 3D backbone [78], ScaLR generalizes to other datasets, e.g., SemanticPOSS [71]. Other authors [61] have retrained it to include WOD [22]. 

ScaLR +

. Although the supervision signal of ScaLR is based on the similarity between pixel and point features, ScaLR, as a general 3D VFM, is not tuned to output 3D features that resemble the most to the 2D features; it is designed and tuned to produce good general 3D features, which are actually extracted before the 3D-2D projection head [78], and thus dissimilar from the 2D features. To get a 3D network that mimics as much as possible the 2D VFM (Tab. 4), we modify the ScaLR setting by: (1) We reintroduce the (linear) projection head and extend it to an MLP; (2) We improve the representation learning by adding a drop path regularization [34] and replacing ReLUs by GELUs [32]; (3) We increase the resolution of images; (4) We train for longer. We call it ScaLR +.

3.2 Prototype generation 

Our second ingredient is a text-conditioned image generator (IG), to generate prototype images from text prompts defining classes of interest (cf. Fig. 3). 

Explicit prompting. Given a set of classes C = {C1, C 2, . . . , C N } to seg-ment, we first build corresponding text prompts. It is common to find broad classes that contain widely different subclasses. For instance, in nuScenes, the 

manmade class is defined in the annotator instructions [62] to contain things as different as building, wall, stairs, pole, fire hydrant, traffic sign and traffic light. 8 N. Samet et al.

It does not matter when training with full supervision, but it does matter when trying to represent a class with a single prototype. Therefore, contrary, e.g., to the bicycle class, which can easily be defined by just one prototype, the manmade 

class has to be split into different subclasses, defined with separate prototypes, as all other 3D OVSS methods also do. Such subclasses {ci, 1, c i, 2, . . . , c i,N i } are mapped to their corresponding class Ci at a later stage, for evaluation. This ap-plies as well to “negative” classes, e.g., other flat in nuScenes, which we actually define as traffic island , following again the nuScenes’ annotator instructions [62]. 

Focus on objects and stuff. When asked to generate an image picturing an object, image generators tend to include a background, which may confuse the extraction of object-specific features. As a workaround, OVDiff [42] uses CutLER [100] to discard the background. An alternative is to use SAM [6,45,81], which is heavier and requires prompting. We prefer to ask the IG to generate, for 

things , the object in a white background, and, for stuff , some stuff covering the whole image. For each subclass ci,j of the thing (resp. stuff ) type, we create a text prompt ti,j of the form “Generate an image of [CLASS] with a white background ”(resp. “ covering the whole image ”) where [CLASS] is the name of ci,j .After we generate an image for a subclass ci,j , we apply a tight cropping around the object to remove the empty (white) areas at the borders of the pic-ture, resulting in a prototype image Ii,j . To obtain the corresponding prototype feature f 2D  

> i,j

, we first normalize the individual patch features extracted from the encoded prototype image Ii,j ; we then compute their average and normalize the resulting feature vector. In driving datasets, cameras and lidar have similar viewpoints, hence “see” similar occlusions. During distillation, the 3D network is trained to produce point features similar to 2D patch features, regardless of occlusions. As patch features tend to be similar within an object, it justifies taking the average as prototype.

3.4 IGLOSS variant 1: closed-set self-training 

The above method, which we call IGLOSS, is open-vocabulary at inference time. It is largely free from biases outside of those present in its 2D foundation models and distillation sources. Consequently, it can segment arbitrary classes given by free text prompts without retraining a 3D model, e.g., for a interactive retrieval. But when the classes are fixed, e.g., for auto-labeling, it is possible, after label assignment, to complement IGLOSS with a self-training phase [122] to improve the label predictions. For this closed-set self-training , denoted IGLOSS clo , we first improve the 4D (spatial and temporal) consistency by aggregating each sequence of labeled scans into the same coordinate system and enforcing label consistency by majority voting of the points falling into a voxel [85]. We then propagate voxel classes back to points of individual scans, and consider them as pseudo-labels to finetune the distilled 3D backbone of the 3D VFM (cf. Sec. 3.1). 

3.5 IGLOSS variant 2: 2D-3D VLM mixing 

If several 2D-3D distillations are available and complementary with respect to their features, it is profitable to combine them. In particular, it is possible to 10 N. Samet et al. 

make a single 3D distillation of concatenated 2D features, but the resulting high-dimensional mixed features can be difficult for a 3D network to regress. Alternatively, we just concatenate the feature vectors of the 2D VLMS to get images features, and of the corresponding 3D distillations to get point features. 

# 4 Experiments 

In this section, we provide implementation details (Sec. 4.1), quantitative re-sults and comparisons, both in the open-set open-vocabulary setting and the annotation-free closed-set setup (Sec. 4.2), as well as ablation studies (Sec. 4.3). 

4.1 Implementation Details Lidar datasets. We conduct our experiments on classical automotive datasets, nuScenes (NS) [5] and SemanticKITTI (SK) [1], with identical parameters. 

Aligned 2D-3D VFMs. We use a ScaLR + (Sec. 3.1) with a multi-dataset [1,5,105] distillation of DINOv2 [68]. A single model can thus be shared across NS and SK, in contrast to prior work that trains separate models for each dataset. 

Image generation. We generate or get prototype images from four sources: ChatGPT-5 (Plus) [67], Gemini 2.5 (Pro) [27], Flux [46] and Google Image Search. For Web-based collection, we query images with transparent backgrounds to be consistent with generated images (Sec. 3.2). Fig. 3 shows image examples for nuScenes classes. For SemanticKITTI, we use the same set of images as for nuScenes, generating additional images only for classes that are not in nuScenes; specifically, we add images for motorcyclist , bicyclist , and parking .

Feature extraction and classification. We resize prototype images to 448 × 448 and extract DINOv2 features at the layer used for 2D-3D distillation. Patch features are averaged into a single 1024-d feature vector per prototype. (We tried a few alternatives. Keeping all patch features as individual prototypes does not improve performance. We also evaluated CutLER [100], used in OVDiff [42], to discard background patches before either feature averaging or prototyping with individual patches, but foreground masks were inaccurate and did not help.) Logictic regression uses liblinear solver [23] with default regularization C = 1.

IGLOSS variants. Unless otherwise stated, IGLOSS is our method with a ScaLR + distillation of DINOv2 in WaffleIron-48-768 [77], two ChatGPT images per subclass prototype (prompt), and classification with logistic regression (LR). We denote by IGLOSS clo the closed-set model after self-training with labels given by IGLOSS (cf. Sec. 3.4). Voxel-based point consistency operates with voxels of 10 cm. We then finetune ScaLR + with these pseudo-labels as done in ScaLR [78], for 10 epochs, with a learning rate of 10 -4 and a layer-wise learning rate decay of 0.99. While IGLOSS is OV at inference, IGLOSS clo is not. We call IGLOSS mix our method applied to a mix (Sec. 3.5) of ScaLR + distilled from DINOv2 and OpenScene distilled from OpenSeg [74]. In this section, we present ablation studies that justify our design choices. For efficiency, unless otherwise mentioned, we evaluate in simplified settings, using 

only one image per prompt and, sometimes, a reduced nuScenes dataset. This explains the differences with the “final” results reported in Tab. 2. 

2D VFM evaluation. We evaluate 2D VFMs for their ability to label lidar scans by unprojecting image labels onto 3D points. We compare OVDiff, that also uses image generation, SigLIP 2 [94], OpenSeg [28], that is the strongest image feature extractor used with OpenScene [74], and DINOv2. Results in Tab. 3(a) show that DINOv2 yields the best performance. It confirms that current CLIP-like 2D VLM features are not as strong as pure 2D VFM features, likely because of the training additionally needs to align with text features too, which also limits training data to captioned images. In contrast, a 2D VFM like DINOv2, which does not need any training annotation, can exploit any image. It also shows that OVDiff with StableDiffusion is not as strong as DINOv2 with ChatGPT. 

2D-3D VFM distillation evaluation, NN vs LR. Results above in unprojecting 2D VFM knowledge however do not prejudge the quality of 2D-3D distillation, which we evaluate here. # 5 Conclusion 

IGLOSS is a novel approach for 3D OVSS that evades the need for text and visual data to share the same feature space. It combines the use of image generation (IG) with a well-crafted 2D-3D distillation method, ScaLR +, and defines a new SOTA for both 3D OVSS and annotation-free close-set semantic segmentation. This joint use of IG and aligned 2D-3D VFMs opens new perspectives in OV approaches. The simplicity of IGLOSS and its SOTA performances make it a strong baseline to build on. It also provides a bridge with few-shot methods.

Last, we summarize the main ideas of 3D open-vocabulary semantic segmentation (OVSS) methods that apply to driving data (Sec. A.4) and we compare them to our method (Sec. A.5). 

A.1 Image-text modality gap in VLMs 

Several authors have highlighted [40,53,79,108], via both theoretical analyses and empirical studies, an inherent modality gap between text and images in Visual Language Models (VLMs) like CLIP [80], which try to directly align images and text. One of the main causes is the disparity of initializations induced by the need to train separate encoders of different nature for each modality [53]. It tends to create a feature gap that contrastive learning, which many VLMs rely on, does not bridge when training [53, 79]. Other reasons include intrinsic goal contradictions for building joint representations of images and text [40]. Besides, analyses of the joint feature space suggest that the uniformity of representations on the hypersphere is also a key aspect that has to be addressed while trying to minimize the gap between image and text features, especially since direct gap minimization may compromise the uniformity of these represen-tations [99, 108]. 

A.2 Addressing the VLM modality gap 

A number of approaches have been proposed to address this modality gap is-sue, at different stages of the training and inference of the VLM: in the VLM pretraining stage (Sec. A.2.1), as a refinement coming after VLM pretraining (Sec. A.2.2), in a downstream finetuning stage (Sec. A.2.3), or as a test-time tuning (Sec. A.2.4). IGLOSS: Image Generation for Lidar OVSS 23 

A.2.1 VLM pretraining. When pretraining the text and image encoders, the feature space alignment can be improved with additional or alternative loss terms [29,48], textual negative engineering [18], augmentation awareness [48], or self-supervised learning training recipes [51]. Another direction is to introduce some sharing at architecture level between image and text, e.g., sharing the weights of a single transformer after a first separate embedding stage [110], or sharing tokens [12]. Although these approaches contributes to a better feature alignment, a modal-ity gap remains. Besides, these approaches involve full pretraining, which requires large-scale image-text pair datasets and heavy compute to reach good practical performance. 

A.2.2 VLM post-pretraining. A better alignment can also be obtained after finetuning with a loss that tries to match feature distributions rather than directly-paired features, while using the pretrained VLM as teacher to preserve past knowledge [108]. 

A.2.3 VLM finetuning for a downstream task. Another line of work focuses on finetuning for a particular downstream task or dataset, possibly with learnable parameters on top of a VLM with frozen weights. This includes textual prompt learning [117, 118], visual prompt tuning [37] or jointly learning prompts for both modalities [44] or for multiple tasks [89]. Still keeping the visual and textual backbones frozen, multi-modal adapters with shared projection layers, operating between encoder transformer blocks, have been proposed to preserve generalization while improving discrimination for specific tasks or data [109]. Observing that the poor alignment of textual and visual representations, even after finetuning, is partly due to separate embedding spaces, [65] uses a mixup strategy to distribute multi-modal features more uniformly on the unit hypersphere when finetuning. 

A.2.4 Test-time tuning. Last, some approaches only operate at test time. A first category of method tries to optimize the textual prompts, e.g., tuning them at test time based on consistency under augmentations [59] or using a Large Language Model (LLM) to elaborate class descriptions highlighting, at textual level, what characteristics a class is expected to have [60, 76]. A second kind of method operates at image level, using few shots and their VLM visual encoding to adjust classification predictions [95]. In fact, [79] argues, based on theoretical justifications, that it is better for alignment to only operate in the vision space, and use text features only to get pseudo-labels from which vision prototype features can then be obtained. This is what IGLOSS does, although very differently from [79]. 24 N. Samet et al. 

A.3 Levels of openness 

3D OVSS methods are open-vocabulary in that any text prompt can be used to define a semantic class to segment. The time when these text prompts are used and when the semantic concepts are fixed, compared to the amount of processing that follows (training for distillation, inference, etc.), however widely varies. There is in fact a hierarchy of openness w.r.t. vocabulary and semantic con-cepts: from closed-set models trained with open, prompt-based labels (marked 

✗ in Tab. 1), to models semantically biased by input text prompts but still OV (marked ±±±), to OV models possibly semantically biased by the power of an un-derlying captioner (marked ( ✓)), to fully OV and open-world models (marked 

✓). IGLOSS falls in that latter category ( advantage #7 ). See however Sec. A.3.5 for an intrinsic remaining bias of all methods, origi-nating from necessary 2D-3D distillation. 

A.3.1 3D training from labels. A first kind of 3D OVSS methods [10, 26, 41,85,92,120] starts by predicting 2D semantic labels from text prompts via the 2D VLM, and trains the 3D network with these pseudo-labels. These image-labeling-time OV methods are open-vocabulary before training time, but not anymore at test time, when they are closed-set as they can then only segment the previously-defined classes. 

A.3.2 3D training with the help of text prompts. A second kind of methods [11, 52, 98] uses given text prompts of classes of interest early in the process too, but only to guide 2D-3D distillation, which is then performed on 2D VLM features rather than on semantic labels. These training-time OV methods are open-vocabulary in the sense that the 3D network produces features that can be compared at test time to the VLM features of any text, not just of the text prompts provided earlier. However the training is biased towards the closed set of previously-defined classes. The method may thus not fully be open regarding the capacity to recognize semantic concepts beyond these known classes. 

A.3.3 3D training with the help of a captioner. A third kind of methods [38, 101] also exploits some text prompts early, but these texts are obtained automatically via an image-to-text model that describes with words images in the dataset, rather than being provided by the user. As is the case for the second category of methods above (Sec. A.3.2), these descriptions are then used as text prompts to guide 2D feature distillation onto 3D. Again, the 3D network then produces features that can be matched to the feature of any text prompt. Such auto-vocabulary OV methods are thus open-vocabulary at interface level. Yet, the training is somehow biased towards the kind of classes that could be identified automatically at image description time. These methods are more open than training-time 3D OVSS methods, but still possibly not entirely open to all semantic concepts present in the VLM (pre)training dataset. IGLOSS: Image Generation for Lidar OVSS 25 

A.3.4 3D training from 2D features only. A fourth and last kind of method [69,74] makes no assumption regarding semantic classes that may appear in the distillation training set and at test time. Existing such test-time OV methods directly distill 2D VLM features into a 3D network, which produces 3D features that can be compared to any text features. These methods are not only open-vocabulary at test time; they are also fully open in terms of semantic concepts, within the limits of the concepts appearing in the VLM pretraining dataset and in the 2D-3D distillation dataset (Sec. A.3.5). Our approach differs in that we only align visual features, i.e., 2D and 3D features. Free text prompts are then used to generate images from which 2D features are extracted and matched to 3D points. IGLOSS is thus also a test-time 3D OVSS method in the above sense. 

A.3.5 The remaining distillation bias. Due to the lack of enough 3D data paired with text to directly train a decent 3D OV model, images have to be used as an intermediate modality to construct 3D OVSS, via 2D-3D knowledge distillation. It induces a somewhat neglected bias on the dataset used for this distillation, especially since the quantity of aligned 2D-3D data available is also small compared to the amount of data a 2D VLM is trained on. Indeed, resulting distilled 3D networks are not as open as the 2D VLM or VFM they are derived from: only semantic concepts that are present in the target dataset and identified by the 2D VLM (by producing corresponding features) are distilled onto 3D. For instance, while a car or a bookshelf can be easily identified by a 2D VLM, it is likely that the 3D point cloud of a car (resp. a bookshelf) could not be recognized as such by a 3D network distilled from that same 2D VLM if based on an indoor (resp. outdoor) dataset. This is however not an issue in practice, as long as the training dataset is representative enough of the test data. IGLOSS actually benefits from a 3D VFM that is distilled from several datasets [78], which improves its generalization capability. On another note, a limitation also comes from the fact that the correspon-dence between 3D points and text is indirect, as it is constructed via 2D visual features. It is thus also sensitive to noise in 2D-3D correspondences, e.g., due to issues in sensor calibration, sensor synchronization or sensor parallax. 

A.4 3D OVSS methods applied to driving data 

We describe in this section the main ideas of 3D OVSS papers that present appli-cations to lidar data in driving settings. The presentation is organized according to their level of vocabulary and semantic concept openness (Sec. A.3). 

A.4.1 Image-labeling-time 3D OVSS methods. Image-labeling-time 3D OVSS methods use text prompts to pseudo-label images before the 3D network is trained, and are thus closed-set at test time. 26 N. Samet et al. 

CNS [10] refines CLIP-based [116] dense pseudo-labels on SAM [45] masks, and uses them to train both a 2D and a 3D network. A second training stage for regularization includes self-training and cross-training (between 2D and 3D) as well as feature alignment with SAM. 

HICL [41] trains a 3D model from 2D pseudo-labels generated by MaskCLIP [116] using a contrastive loss involving geometric consistency given by super-points [72] and correlation learning, both intra- and inter-scenes. 

AdaCo [120] obtains image masks from FastSAM [115], which are labeled automatically with text using SSA [9]. These text labels are then matched with text prompts using word2vec [13]. Unprojected 3D semantic labels are denoised by voxelwise voting across a short sequence of frames. Two training recipes reduce the impact of pseudo-labeling noise: one based on the learning curve, and another one on a clustering of instances using DBSCAN [21]. The whole pipeline is trained with a robust loss. 

LeAP [26] combines Grounding DINO [57] and SAM [45], similarly to what GSAM [82] does, to get image pseudo-labels from text prompts. After unpro-jection onto 3D, spatial and time consistency is enforced based on 2D class probabilities and on a 3D network trained on the most confident pseudo-labels. 

LOSC [85] denoises 2D pseudo-labels from OpenSeeD [112], enforcing ro-bustness under augmentations and consistency through time at voxel level after unprojection onto 3D. A 3D backbone is then self-trained using iterative fine-tuning from a pretrained ScaLR network [78]. 

A.4.2 Training-time 3D OVSS methods. Training-time 3D OVSS meth-ods use given text prompts of classes of interest not only at test time but also to guide 2D-3D distillation. They are thus biased towards these classes despite being functionally open-vocabulary. 

CLIP2Scene [11] trains a 3D network with a contrastive loss to align point features to CLIP features of text prompts defining classes of interest, via point-pixel correspondences and MaskCLIP 2D features [116]. 

GGSD [98] distills CLIP into 3D using correspondences of pixels with both points and unsupervised superpoints, maximizing cosine similarity. A second training stage adds a contrastive self-distillation loss term that exploits the pre-dictions of the 3D network, based on given text prompts. 

AFOV [92] pretrains its 3D network using image masks from SAN [106], which is CLIP-based, using a fixed class dictionary. AFOV turns these masks into superpixels and corresponding superpoints, and uses a contrastive loss on superpoints involving CLIP features of both superpixels and text. The network is then trained on SAN pseudo-labels. Additionally, AFOV uses a non-parametric module for label denoising. It also possibly exploits images at test time to slightly improve performance. 

SAS [52] ensembles several VLMs, possibly of different nature, e.g., LSeg [50], which is CLIP-based, or OpenSeg [28] and SEEM [121], which are not. To that end, SAS uses TAP [70] as a captioner to generate descriptions for each mask of each VLM, to enrich the initial labels with more semantic information. shared text encoder, like CLIP’s, then aligns the visual information of different VLMs via the encoded rich descriptions. To ensemble information of the different VLMs, SAS relies on a pre-built vocabulary consisting of common classes in the scene (e.g., car, bicycle). For each of these classes, Stable Diffusion (SD) [83] is used to generate several images. A.4.3 Auto-vocabulary 3D OVSS methods. Auto-vocabulary 3D OVSS methods leverage a separate VLM that provides text descriptions from images and use these descriptions to drive the 2D-3D distillation. 

OV3D [38] gets rich image descriptions using an Large Visual Language Model (LVLM), namely LLAVA [55]. These descriptions are then used to pseudo-label image masks from SEEM [121], using SEEM as well for text-image align-ment. The 2D pseudo-labels are then unprojected onto 3D for distillation into a 3D backbone on a target dataset. OV3D may also be combined with OpenScene-3D [74] for a slightly better performance. 

3D-AVS [101] focuses on text generation to automatically label segments. It uses existing CLIP-aligned encoders for text (CLIP [80]), image (OpenSeg [28]) as well as points (OpenScene [74] w/ OpenSeg [28]), trained on the same target dataset. 3D-AVS first trains on the target dataset a point-cloud captioner from an image captioner (BLIP-3 [107]). Given a point cloud, this captioner generates text tags that are used for similarity-based 3D segmentation. At test time, these tags are mapped via LAVE [96] into predefined classes using a Large Language Model (LLM), namely GPT-4o [66]. 

A.4.4 Test-time 3D OVSS methods. Test-time 3D OVSS methods pre-serve the openness of both the vocabulary and the corresponding semantic con-cepts until evaluation time. 

OpenScene [74] distills a CLIP-like model (LSeg [50] or OpenSeg [28]) into a 3D network on a target dataset, maximizing the similarity of 2D and 3D features of short sequences. While inference on pure 3D data is possible, the best (official) results are obtain by ensembling 2D and 3D alignment scores with text, requiring images at test time. 

SAL [69] gets 3D masks from unprojected SAM [45] image masks, which are assigned MaskCLIP features [16] and refined via DBSCAN clustering. A 3D backbone is trained on a target dataset to predict MaskCLIP features as well as SAM masks. 28 N. Samet et al. 

A.5 Comparing 3D OVSS methods for driving data 

Tab. 1 lists 3D OVSS methods that apply to driving lidar data and summarizes their main characteristics. 

A.5.1 Image-text alignment. Almost all of these methods rely on a 2D VLM, mostly CLIP [80] or one of its dense variants [16, 116]. This 2D VLM provides an alignment between image ( i) and text ( t), via a shared feature ( f )space: “ i ⇒ f ⇐ t ”. However, this architecture restrains the alignment quality and requires a VLM trained on a large amount of curated data or extra supervi-sion. IGLOSS does not need such a VLM to bridge image and text ( advantages #1 and #2 in Sec. 1). Moreover, the training architecture of CLIP makes it difficult to obtain good segment boundaries [16, 17, 50, 116]. An auxiliary foundation model for mask (m) segmentation (“ i ⇒ m ”), such as SAM [45], is thus often used to generate masks and learn with segments to improve the quality of final segmentation [10, 26, 52, 69, 114, 120]. In contrast, IGLOSS leverages a 2D Vision Foundation Model (VFM) good at dense tasks ( advantage #3 ), namely DINOv2 [68], which incidentally is trained with no supervision contrary to SAM. Besides, the VFM is used in a black box manner, which facilitates modularity and improvement of IGLOSS when a better VFM becomes available ( advantage #4 ). Only AdaCo [120] and 3D-AVS [101] differ in global architecture. Rather than directly extracting VLM features of image masks (“ i ⇒ f ”), AdaCo first generates textual descriptions of masks with a captioner (“ i ⇒ t ”) [9], and com-pares these descriptions to the given text prompts via word2vec [13] (“ t ⇒ f ”for both texts). As for 3D-AVS, it uses generated captions similarly, but from a trained point captioner and still relying on an underlying VLM. (Some other methods use captioners too, but for other purposes, such as text enrichment [52] or auto vocabulary [38].) In contrast, IGLOSS needs neither encoders aligned both on image and text (“ i ⇒ f ⇐ t ”), nor captioners (“ i ⇒ t ”). It bridges image and text “in the opposite direction”: using an image generator conditioned on text (“ t ⇒ i ”). The alignment of points and text then just comes from the 2D-3D distillation (“ i ⇒ f ⇐ p ”) of a 2D VFM. At test time, no image is needed ( advantage #6 ); we only have to compare the features of the point clouds (“ p ⇒ f ”) to the fea-tures of the generated images (“ t ⇒ i ⇒ f ”). 

A.5.2 Simplicity. Another difference of IGLOSS with most other methods is that it is extremely simple ( advantage #5 ) and does not require any training, beyond the direct 2D-3D feature distillation, for which there are proven recipes such as ScaLR [78], and even pretrained 3D network readily available [78]. The image generator (“ t ⇒ i ”) is used as a complete black box ( advantage #4 ), con-trary to SAS that uses Stable Diffusion only as a way to get cross-attention maps and extract intermediate masks. In contrast, many other methods use a heavy machinery relying on many modules and possibly involving several train-ing stages. For instance, SAS [52] and 3D-AVS [101] leverage simultaneously IGLOSS: Image Generation for Lidar OVSS 29 

seven underlying external tools, and CNS [10], CLIP2Scene [11], GGSD [98], AdaCo [120] and SAS train in two phases, while LOSC [85] performs several self-training iterations. 

A.5.3 Vocabulary and semantic concept openness. As discussed above, there are different levels of vocabulary and semantic concept openness for 3D OVSS methods, which may come come with biases (Sec. A.3). An extra bias may also come from the 2D-3D distillation (Sec. A.3.5). In fact, all 3D OVSS methods actually use the target (evaluation) dataset to train the distillation in the first place. In contrast ( advantage #7 ), IGLOSS is fully open-vocabulary and it uses a foundation model that is trained on several datasets [78], which reduces the bias, although it does not eliminate it entirely. 

A.5.4 Use of images or scan sequences at test time. Some 3D OVSS methods [74, 92, 101] propose to also use images at test time to improve perfor-mance. The underlying 2D VLM then provides soft labels [92] or features [74,101] for ensembling with 3D information, after unprojection. It has however been ar-gued [98] that 2D-3D ensembling at test time has high storage and computational costs, especially since a lidar frame is generally covered by several cameras (e.g., 6 in nuScenes). Other methods exploit a sequence of scans [74, 101, 120] rather than a single frame, merging them to increase point density. IGLOSS needs neither images nor scan sequences at test time ( advantage #6 ) to get SOTA results (Tab. 2). 

A.5.5 2D VLM ensembling. While OV3D [38] proposes an integration with OpenScene-3D [74], SAS [52] is specifically designed to ensembles VLMs, possibly of different nature. Ensembling is not required in IGLOSS ( advantage #10 ) to get SOTA results (Tab. 2). It is however particularly easy to implement (see Sec. 3.5) and can further boost results by 3.6 to 4.6 mIoU pts ). 

A.5.6 Datasets used for evaluation. As visible in Tab. 2, all above methods (Sec. A.4) evaluate on the nuScenes dataset [5], which features a rotating lidar and six cameras around the ego-vehicle, covering together a 360 ° field of view. However, only very few of them evaluate on SemanticKITTI [1]. One of the reasons could be that this dataset also features a rotating lidar, but just one front camera. Only information from a single front view can thus be distilled into a whole 3D scan, which makes it more difficult for the distilled network to generalize to unseen areas on the side and on the back of the ego-vehicle. Again (Sec. A.5.3), IGLOSS suffers less from this situation as it leverages a 3D VFM [78] that is trained from several datasets at the same time [1, 5, 105].
