## 摘要
多模态大语言模型（MLLMs）通过提高输入保真度来获得更强的视觉理解能力，但由此产生的视觉 token 增长使得同时维持高空间分辨率和长时间上下文变得成本极高。我们认为，瓶颈不在于编码后的表示如何压缩，而在于编码器接收的像素量。为此，我们提出了 ResAdapt，这是一种输入端适配框架，用于学习在编码前每帧应分配多少视觉预算。ResAdapt 将轻量级分配器（Allocator）与未经修改的 MLLM 主干网络相结合，使主干网络在接收算子转换后的输入时，仍能保留其原生的视觉 token 接口。我们将分配过程建模为上下文老虎机（contextual bandit），并使用成本感知策略优化（CAPO）来训练分配器，该方法将稀疏的 rollout 反馈转化为稳定的准确度-成本学习信号。在预算受限的视频问答、时间定位和图像推理任务中，ResAdapt 改善了低预算运行点，且通常处于或接近效率-准确度前沿，在激进压缩下的推理密集型基准测试中提升最为显著。值得注意的是，在相同的视觉预算下，ResAdapt 支持多达 16 倍的帧数，同时带来超过 15% 的性能提升。

## Abstract
Multimodal Large Language Models (MLLMs) achieve stronger visual understanding by scaling input fidelity, yet the resulting visual token growth makes jointly sustaining high spatial resolution and long temporal context prohibitive. We argue that the bottleneck lies not in how post-encoding representations are compressed but in the volume of pixels the encoder receives, and address it with ResAdapt, an Input-side adaptation framework that learns how much visual budget each frame should receive before encoding. ResAdapt couples a lightweight Allocator with an unchanged MLLM backbone, so the backbone retains its native visual-token interface while receiving an operator-transformed input. We formulate allocation as a contextual bandit and train the Allocator with Cost-Aware Policy Optimization (CAPO), which converts sparse rollout feedback into a stable accuracy-cost learning signal. Across budget-controlled video QA, temporal grounding, and image reasoning tasks, ResAdapt improves low-budget operating points and often lies on or near the efficiency-accuracy frontier, with the clearest gains on reasoning-intensive benchmarks under aggressive compression. Notably, ResAdapt supports up to 16x more frames at the same visual budget while delivering over 15% performance gain. Code is available at https://github.com/Xnhyacinth/ResAdapt.

#### 1. 核心问题与整体含义（研究动机和背景）
多模态大语言模型（MLLM）在处理视频推理时面临**空间分辨率**与**时间覆盖范围**的矛盾：提高分辨率或增加帧数都会导致视觉 Token 数量爆炸，超出计算预算。
*   **现有方法的缺陷**：
    *   **模型侧压缩**（如 Token 剪枝/合并）：在编码器运行后才干预，计算成本已产生，且破坏了推理引擎的优化内核。
    *   **输出侧代理**（如迭代检索/缩放）：引入多轮交互延迟，且初始粗略视图可能漏掉关键线索。
*   **研究动机**：在**编码前**（输入侧）动态调整每帧的分辨率，将有限的像素预算分配给信息最密集的帧，从而在不增加计算负担的前提下提升长视频推理能力。

#### 2. 论文提出的方法论
ResAdapt 框架将视觉预算分配建模为一个**上下文老虎机（Contextual Bandit）**问题。
*   **核心组件**：
    *   **分配器（Allocator）**：一个轻量级的 Transformer 模块，接收粗略的视觉特征和文本查询，预测每帧的缩放因子 $s_t$。
    *   **算子（Operator）**：根据分配器的预测对原始图像进行双线性插值缩放，随后送入 MLLM 主干。
*   **关键技术：成本感知策略优化（CAPO）**：
    *   **动态成本基准（Dynamic Pivot）**：结合群体均值和全局目标预算，动态决定当前分配是应受奖励（高效）还是惩罚（昂贵）。
    *   **非对称奖励塑造**：对“正确且低成本”的回答给予适度奖励，对“错误且高成本”的回答给予严厉惩罚，防止策略坍缩至最小预算。
    *   **时间相似性正则化（$L_{sim}$）**：惩罚相邻相似帧同时分配高预算的行为，强制模型在冗余帧间进行差异化选择。
*   **算法流程**：分配器生成 Beta 分布参数 $\rightarrow$ 采样缩放因子 $\rightarrow$ 调整图像大小 $\rightarrow$ MLLM 推理 $\rightarrow$ 根据准确度和成本通过 RL（如 GRPO 算法）联合更新分配器和主干网络。

*   **数据集与场景**：
    *   **视频问答（Video QA）**：VideoMME, LongVideoBench, MMVU, MLVU, VideoMMMU, LVBench。
    *   **时间定位（Temporal Grounding）**：Charades-STA, ActivityNet, NExT-GQA。
    *   **图像理解**：MathVista, MMMU, OCRBench, ChartQA 等。
*   **主干网络**：Qwen2.5-VL-7B, Qwen3-VL-8B。
*   **对比方法**：随机丢帧（Random Drop）、固定缩放（FixedScale）、Token 合并（ToMe）、VisionZip、FlashVid 以及推理时增强方法 VideoAuto-R1。

#### 4. 资源与算力
*   **硬件环境**：使用了 **32 台 NVIDIA H100 GPU**。
*   **训练细节**：基于 VeRL 框架和 vLLM 推理引擎，采用 DeepSpeed 进行优化。训练周期为 1 个 Epoch，全局 Batch Size 为 128。
*   **分配器开销**：分配器（基于 SmolVLM 架构）的计算量仅占总推理 FLOPs 的不到 3%，在长视频场景下其开销可被主干网络的节省完全抵消。

#### 5. 实验数量与充分性
*   **实验规模**：论文在超过 15 个主流基准测试上进行了评估，涵盖了从短视频到长视频（128 帧）、从感知到复杂推理的多种维度。
*   **消融实验**：详细探讨了 CAPO 奖励设计、时间正则化项的作用、不同分布函数（Beta vs. Normal）的影响以及算子迁移性（缩放迁移至帧选择）。
*   **客观性**：通过跨模型（Qwen2.5/3）验证和零样本迁移实验，证明了分配策略的泛化能力，实验设计较为全面且对比公平。

#### 6. 主要结论与发现
*   **效率飞跃**：ResAdapt 在消除超过 90% 视觉 Token 的情况下，依然能匹配或超过全分辨率基准。
*   **时间扩展**：在相同计算预算下，ResAdapt 可处理 **16 倍**更多的帧数，在长视频推理任务中获得超过 15% 的相对增益。
*   **主动感知**：模型自发学会了将预算集中在关键帧（如文字出现、场景切换、动作瞬间），这种能力完全通过任务奖励习得，无需显式显著性标注。

#### 7. 优点
*   **输入侧适配**：在编码前干预，真正节省了视觉编码器的计算开销。
*   **硬件友好**：生成的 Token 序列是标准的，无需修改 FlashAttention 或 vLLM 等底层内核即可享受加速。
*   **RL 稳定性**：CAPO 有效解决了强化学习在处理连续动作空间和成本惩罚时的策略坍缩问题。

#### 8. 不足与局限
*   **短视频收益有限**：对于短序列（$\le 32$ 帧），分配器的固定开销可能抵消部分加速效果。
*   **开环决策**：分配器在推理前一次性做出决定，无法在推理过程中根据中间结果动态修正（例如发现某帧模糊后重新高精度编码）。
*   **跨模态差异**：虽然在视频上表现优异，但在纯图像任务上的表现提升不一，对极小目标的捕捉仍存在漏掉关键线索的风险。
