## 摘要
大型基础模型为自主系统提供了强大的推理能力，但将语义意图映射到可靠的实时控制仍具挑战。现有方法要么 (i) 让大语言模型 (LLM) 直接生成轨迹——这种方式脆弱、难以验证且易产生延迟；要么 (ii) 在线调整模型预测控制 (MPC) 的目标——这混合了慢速深思熟虑与快速控制，模糊了接口界限。我们提出了“智能体快慢规划”（Agentic Fast-Slow Planning），这是一个在自然时间尺度上解耦感知、推理、规划和控制的分层框架。该框架包含两个桥梁：Perception2Decision 使用车载视觉语言模型 (VLM) 检测器将场景压缩为以自我为中心的拓扑结构，然后利用云端 LLM 决策器将其映射为符号化驾驶指令，在保持可解释性的同时减少了带宽和延迟；Decision2Trajectory 将指令转换为可执行路径，其中语义引导的 A* 算法将源自语言的软成本嵌入经典搜索中，使解偏向可行轨迹，而智能体细化模块则利用反馈和记忆来调整规划器的超参数。最后，MPC 实时跟踪轨迹，并针对困难情况提供可选的云端引导参考。在 CARLA 中的实验表明，与纯 MPC 和 A* 引导的 MPC 基准相比，智能体快慢规划提高了扰动下的鲁棒性，横向偏差减少了高达 45%，完成时间缩短了 12% 以上。代码可在 https://github.com/cjychenjiayi/icra2026_AFSP 获取。

## Abstract
Large foundation models enable powerful reasoning for autonomous systems, but mapping semantic intent to reliable real-time control remains challenging. Existing approaches either (i) let Large Language Models (LLMs) generate trajectories directly - brittle, hard to verify, and latency-prone - or (ii) adjust Model Predictive Control (MPC) objectives online - mixing slow deliberation with fast control and blurring interfaces. We propose Agentic Fast-Slow Planning, a hierarchical framework that decouples perception, reasoning, planning, and control across natural timescales. The framework contains two bridges. Perception2Decision compresses scenes into ego-centric topologies using an on-vehicle Vision-Language Model (VLM) detector, then maps them to symbolic driving directives in the cloud with an LLM decision maker - reducing bandwidth and delay while preserving interpretability. Decision2Trajectory converts directives into executable paths: Semantic-Guided A* embeds language-derived soft costs into classical search to bias solutions toward feasible trajectories, while an Agentic Refinement Module adapts planner hyperparameters using feedback and memory. Finally, MPC tracks the trajectories in real time, with optional cloud-guided references for difficult cases. Experiments in CARLA show that Agentic Fast-Slow Planning improves robustness under perturbations, reducing lateral deviation by up to 45% and completion time by over 12% compared to pure MPC and an A*-guided MPC baseline. Code is available at https://github.com/cjychenjiayi/icra2026_AFSP.

### 2. 方法论：核心思想与关键技术
AFSP 框架通过两个关键“桥梁”实现了感知、推理、规划与控制的解耦：

*   **Perception2Decision（感知到决策）**：
    *   **车载 VLM 拓扑检测器**：在车端运行轻量化视觉语言模型（如 InternVL-2B），将原始图像压缩为“以自我为中心”的拓扑图（包含物体类别、距离、方位等符号信息），极大降低了上传云端的带宽需求。
    *   **云端 LLM 决策器**：利用 DeepSeek-V3 等强力模型，基于拓扑图进行思维链（CoT）推理，输出符号化的驾驶指令序列（如 `LEFT`, `KEEP`, `RIGHT`）及驾驶风格。
*   **Decision2Trajectory（决策到轨迹）**：
    *   **语义引导 A* 算法**：这是本文的核心创新。它将 LLM 的符号指令转化为 A* 搜索中的**软成本（Soft Costs）**。公式上，在传统几何成本 $g(s)$ 基础上增加了语义成本 $c_{sem}$，通过奖励符合指令的行为、惩罚违背指令的行为，引导搜索算法生成既符合语义意图又具备几何可行性的路径。
    *   **智能体细化模块（Agentic Refinement）**：利用 LLM 作为一个闭环调节器，根据规划反馈和历史记忆自动调整 A* 的超参数（如各项成本权重），替代了繁琐的人工调参。
*   **实时控制**：采用 **云端引导 MPC**，根据场景复杂度动态切换局部参考路径或云端全局路径，确保实时避障与稳定性。

### 3. 实验设计
*   **实验环境**：使用 **CARLA** 自动驾驶仿真器。
*   **数据集**：构建了一个包含 49 类物体、约 20 万帧图像的大规模数据集（涵盖 Town2-7 和 Town10HD），用于微调车载 VLM。

*   **训练算力**：使用了 **4 台 NVIDIA A100 GPU** 进行 VLM 的微调。

*   **实验规模**：

*   **充分性评价**：实验设计较为充分。不仅验证了感知端的准确性，还通过消融实验展示了语义引导 A* 在地图扰动下的鲁棒性。对比指标涵盖了完成时间、轨迹长度、平均/最大横向偏差、速度波动等多个维度，结果客观。

### 6. 主要结论与发现
*   **性能提升**：相比纯 MPC，AFSP 将**最大横向偏差降低了约 45%**，任务**完成时间缩短了 12% 以上**。
*   **鲁棒性**：在地图发生 1-2 米的平移扰动时，传统 A* 容易失效，而语义引导 A* 能够凭借 LLM 的高层意图保持路径的正确性。
*   **效率**：通过拓扑图压缩，系统在保证决策质量的同时，显著降低了云端通信延迟和计算开销。

### 7. 优点与亮点
*   **架构优雅**：成功实现了“快慢系统”的解耦，让 LLM 负责不限时的逻辑推理，让 MPC 负责高频的实时控制，互不干扰。
*   **软约束机制**：将语言指令作为 A* 的“软成本”而非“硬约束”，避免了因 LLM 偶尔误判导致的规划死锁，增强了系统的容错能力。
*   **自动化调参**：引入 Agentic Refinement 模块，利用 LLM 的反思能力自动优化底层算法参数，体现了高度的智能化。

### 8. 不足与局限
*   **云端依赖**：虽然优化了带宽，但系统仍高度依赖云端连接。在隧道、地下车库等无信号场景下的表现未被充分讨论。
*   **动态环境复杂性**：实验场景虽然包含动态车辆，但在极高密度的博弈型交通流（如无信号灯十字路口抢行）中的表现尚待验证。
*   **实时性瓶颈**：尽管控制是实时的，但 LLM 决策仍需数秒。如果环境变化极快（如突发鬼探头），高层决策可能滞后，此时系统高度依赖底层 MPC 的应急反应。
