## 摘要
固定表示容量是持续学习中的一个基本约束：从业者必须在训练前预估合适的模型宽度，而无法预知数据中包含多少个不同的概念。我们提出了 LACE（Loss-Adaptive Capacity Expansion，损失自适应容量扩展），这是一种简单的在线机制，通过监测自身的损失信号在训练期间扩展模型的表示容量。当持续的损失偏差超过阈值时（表明当前容量不足以处理新遇到的数据），LACE 会向投影层添加新维度，并将其与现有参数共同训练。在合成数据和真实数据的实验中，LACE 仅在领域边界触发扩展（边界精度达 100%，零误报），在初始维度仅为一小部分的情况下，达到了与大型固定容量模型相当的准确率，且生成的适配器维度对性能至关重要（移除所有适配器会导致准确率下降 3%）。我们还通过逐层聚类展示了 GPT-2 激活中的无监督领域分离，揭示了跨层的 U 型可分离性曲线，这为深度网络中的自适应容量分配提供了依据。LACE 无需标签、重放缓冲区或外部控制器，非常适合资源受限环境下的端侧持续学习。

## Abstract
Fixed representational capacity is a fundamental constraint in continual learning: practitioners must guess an appropriate model width before training, without knowing how many distinct concepts the data contains. We propose LACE (Loss-Adaptive Capacity Expansion), a simple online mechanism that expands a model's representational capacity during training by monitoring its own loss signal. When sustained loss deviation exceeds a threshold - indicating that the current capacity is insufficient for newly encountered data - LACE adds new dimensions to the projection layer and trains them jointly with existing parameters. Across synthetic and real-data experiments, LACE triggers expansions exclusively at domain boundaries (100% boundary precision, zero false positives), matches the accuracy of a large fixed-capacity model while starting from a fraction of its dimensions, and produces adapter dimensions that are collectively critical to performance (3% accuracy drop when all adapters removed). We further demonstrate unsupervised domain separation in GPT-2 activations via layer-wise clustering, showing a U-shaped separability curve across layers that motivates adaptive capacity allocation in deep networks. LACE requires no labels, no replay buffers, and no external controllers, making it suitable for on-device continual learning under resource constraints.

#### 1. 核心问题与研究动机
*   **核心问题**：在持续学习（Continual Learning）中，神经网络的“宽度”（即每一层的维度）通常是预先固定的。这导致了一个矛盾：如果初始容量太小，模型无法学习后续的新概念（容量墙）；如果初始容量太大，在学习早期会造成计算资源浪费。
*   **研究动机**：从业者在训练前很难预知数据中包含多少个不同的概念。作者希望提出一种机制，让模型能够根据训练过程中的实时反馈（损失信号），自动检测当前容量是否不足，并按需动态扩展。

#### 2. 方法论
*   **核心思想**：利用训练损失（Loss）作为“容量不足”的信号。当模型遇到无法用现有参数表示的新分布时，损失值会显著升高并保持高位。
*   **关键技术细节**：
    *   **损失尖峰检测**：维护一个损失值的指数移动平均（EMA）作为基准 $\bar{L}_t$。当当前损失 $L_t$ 超过基准的 $\tau$ 倍时，判定为一次“尖峰”。
    *   **确认机制与冷却期**：为了过滤噪声，需要连续 $K$ 个步骤触发尖峰才执行扩展。扩展后进入冷却期 $C$，防止频繁无效扩展。
    *   **容量扩展操作**：当触发扩展时，直接向投影矩阵 $W$ 添加一个新的维度（列/行），并使用小的随机权重初始化。
    *   **联合训练**：新增维度与现有参数在后续的梯度更新中共同训练，无需冻结旧参数。
*   **算法流程**：监控损失 -> 检测持续偏差 -> 触发扩展 -> 增加维度 -> 继续训练。

#### 6. 主要结论与发现
*   **高精度触发**：LACE 在所有实验中均实现了 100% 的领域边界检测精度，无误报。
*   **容量效率**：LACE 仅需 Fixed-Large 一小部分的初始维度，就能达到与其相当的最终准确率。
*   **维度有效性**：消融实验显示，移除动态增加的维度会导致准确率下降 3%，证明这些维度确实学习到了关键特征，而非冗余。
*   **U 型分离曲线**：通过对 GPT-2 的分析发现，中间层领域可分离性最差，深层和浅层较好，这暗示了在深层进行容量扩展可能更有意义。

#### 7. 优点
*   **简单高效**：无需复杂的外部控制器、强化学习或神经架构搜索（NAS）。
*   **无需标签/重放**：完全基于自监督的损失信号，不需要任务标签，也不需要存储旧数据的重放缓冲区（Replay Buffer），极大地节省了内存。
*   **在线学习**：支持在训练过程中实时扩展，非常适合流式数据处理。

#### 8. 不足与局限
*   **遗忘问题**：在分类任务中未观察到明显的灾难性遗忘，但这可能是因为输出层保留了所有类别。在生成式任务（如 LLM 预训练）中，LACE 是否能有效缓解遗忘尚待验证。
*   **超参数敏感性**：损失尖峰检测依赖于阈值 $\tau$ 和窗口 $W$，不同任务可能需要调整这些参数以适应不同的噪声水平。
*   **实验规模**：真实世界实验仅涵盖了 3 个领域，对于更复杂、领域重叠更多的超大规模持续学习场景，其表现仍需进一步观察。
*   **架构限制**：目前主要针对投影层（线性层）的宽度扩展，对于注意力机制等更复杂结构的自适应扩展讨论较少。
