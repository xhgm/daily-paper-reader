## 摘要
在复杂环境中的具身导航中，失败是不可避免的。为了增强韧性，重规划（RP）是一个可行的选择，即允许机器人失败，但其能够调整计划直到成功。然而，现有的重规划方法冻结了自身动作模型，错失了通过升级机器人自身来探索更好计划的机会。为了解决这一局限性，我们提出了自进化重规划（简称 SERP），通过从近期经验中进行运行时学习，实现了从冻结模型向进化模型的范式转变。与通常受限于预定义静态参数的现有模型进化方法不同，我们引入了代理式自进化动作模型，该模型利用带自动微分的上下文学习（ILAD）进行自适应函数调整和全局参数重置。为了实现 SERP 的高效 Token 重规划，我们还提出了图思维链（GCOT）重规划，通过大语言模型（LLM）在蒸馏图上进行推理。广泛的仿真和真实世界实验表明，SERP 在各种基准测试中以更低的 Token 消耗实现了更高的成功率，验证了其在不同环境下的卓越鲁棒性和效率。

## Abstract
Failure is inevitable for embodied navigation in complex environments. To enhance the resilience, replanning (RP) is a viable option, where the robot is allowed to fail, but is capable of adjusting plan until success. However, existing RP approaches freeze the ego action model and miss the opportunities to explore better plans by upgrading the robot itself. To address this limitation, we propose Self-Evolutionary RePlanning, or SERP for short, which leads to a paradigm shift from frozen models towards evolving models by run-time learning from recent experiences. In contrast to existing model evolution approaches that often get stuck at predefined static parameters, we introduce agentic self-evolving action model that uses in-context learning with auto-differentiation (ILAD) for adaptive function adjustment and global parameter reset. To achieve token-efficient replanning for SERP, we also propose graph chain-of-thought (GCOT) replanning with large language model (LLM) inference over distilled graphs. Extensive simulation and real-world experiments demonstrate that SERP achieves higher success rate with lower token expenditure over various benchmarks, validating its superior robustness and efficiency across diverse environments.

### 具身导航的代理式自进化重规划 (SERP) 论文总结

#### 1. 核心问题与整体含义（研究动机和背景）
在复杂的具身导航任务中，机器人因环境不确定性或自身模型缺陷导致失败是常态。现有的重规划（Replanning, RP）方法主要集中在语义层面的目标调整或局部路径的重新生成，但它们通常**冻结了机器人的动作模型参数**。这意味着机器人无法通过“学习失败经验”来升级自身性能，容易在同一物理障碍或约束下反复失败。论文的研究动机在于打破“冻结模型”的限制，提出一种能够通过运行时（Run-time）学习实现**自进化**的重规划框架，使机器人能够通过调整自身内部参数来“征服”而非仅仅“绕过”失败。

#### 2. 方法论：核心思想与关键技术
论文提出了 **SERP (Self-Evolutionary RePlanning)** 框架，由两个协同组件组成：

*   **局部代理式自进化 (Local Agentic Self-Evolution, ASE)：**
    *   **核心思想：** 针对物理执行失败，通过优化动作模型的代价函数参数（如路径跟踪权重、避障权重等）来适应环境。
    *   **关键技术 ILAD：** 结合了**带自动微分的上下文学习 (In-context Learning with Auto-differentiation)**。
        *   **自动微分 (AD)：** 通过反向传播微调参数 $P$，解决局部精确优化问题。
        *   **上下文学习 (IL)：** 当 AD 陷入局部最优或失败时，利用视觉语言模型 (VLM) 分析失败原因，进行全局参数重置或损失函数权重调整。
    *   **参数检索 (RAG)：** 使用检索增强生成技术，从历史经验存储器中提取最接近当前场景的初始参数 $P^{(0)}$。
*   **全局图思维链 (Global Graph Chain-of-Thought, GCOT)：**
    *   **核心思想：** 针对语义失败，将大规模 3D 场景图蒸馏为任务相关的紧凑子图。
    *   **流程：** 利用 CLIP 模型进行高效的语义元素检索，而非直接将全图输入 LLM。LLM 仅在蒸馏后的子图上进行逻辑推理和子任务分解，显著降低了 Token 消耗并减少了幻觉。

#### 3. 实验设计
*   **数据集与场景：** 使用了 **Habitat-Matterport 3D (HM3D)** 数据集，涵盖 6 个大型复杂室内场景（如 00824, 00829 等）。
*   **Benchmark 任务：** 共设计了 180 个任务，分为“搜索任务”和“规划任务”，每个任务在环境不确定性下重复 3 次。
*   **对比方法：**
    *   **SayPlan：** 基于 LLM 和 3D 场景图的经典规划方法。
    *   **AD (DiffTune-MPC)：** 纯基于梯度的参数优化方法。
    *   **NeuPAN：** 现有的端到端模型驱动学习导航方法。
*   **评估指标：** 成功加权路径长度 (SPL)、成功率 (SR)、减少图 Token 比率 (RGTR) 和平均适应轮数 (MAEC)。

#### 4. 资源与算力
*   **硬件环境：** 实验在一台运行 Ubuntu 的工作站上完成，配备了 **Nvidia A6000 GPU**。
*   **模型使用：** 采用了 GPT-4o 和 Qwen2.5-3b/Qwen3-Max 作为推理后端。
*   **训练/运行时间：** 论文提到局部 ASE 的单次进化（1 epoch）在真实机器人上耗时少于 3 秒。具体的离线预训练时长未明确说明，但强调了其“运行时（Run-time）”的实时进化能力。

#### 5. 实验数量与充分性
*   **实验规模：** 进行了 180 个仿真任务和多组真实世界机器人实验（涉及办公室、走廊等杂乱环境）。
*   **消融实验：** 详细对比了“仅使用 AD”、“仅使用 IL”以及“ILAD 结合”的效果，验证了两者互补的必要性。
*   **客观性：** 实验考虑了参数偏差（0%-90% 偏差）对模型恢复能力的影响，并引入了物理障碍（如移动的箱子、半开的门）来模拟真实世界的不确定性，实验设计较为全面且贴近实战。

#### 6. 主要结论与发现
*   **性能提升：** SERP 在规划任务中的成功率比 SayPlan 高出 10% 以上，且在 Token 消耗上降低了 90% 以上（RGTR 指标）。
*   **自进化优势：** 相比于纯梯度下降的 AD 方法，SERP 的 ILAD 机制在初始参数极差的情况下，成功率高出 40% 以上，证明了 LLM 介入参数重置能有效跳出局部最优。
*   **跨层协同：** 局部 ASE 处理物理通过性问题，全局 GCOT 处理语义逻辑错误，两者的结合使机器人能应对更复杂的复合型失败。

#### 7. 优点
*   **范式创新：** 提出了从“冻结模型”到“进化模型”的转变，赋予了机器人运行时自我修正的能力。
*   **效率极高：** 通过 CLIP 检索和图蒸馏，解决了 LLM 处理大规模 3D 场景图时的 Token 爆炸和推理延迟问题。
*   **鲁棒性强：** ILAD 机制巧妙地结合了符号推理（LLM）和数值优化（AD），兼顾了直觉理解和精确控制。

#### 8. 不足与局限
*   **搜索精度风险：** 在物体极度密集的场景中，GCOT 的短时记忆机制有时会遗漏搜索目标。
*   **依赖后端模型：** 实验显示使用更强的 LLM（如 GPT-4o）对成功率有显著影响，这意味着系统性能受限于底层大模型的推理能力。
*   **实时性挑战：** 虽然单次优化较快，但在极度动态的环境中，频繁的重规划和参数调整可能仍会带来感知上的停顿。
