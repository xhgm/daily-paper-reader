## 摘要
视觉-语言-动作（VLA）模型在具身智能领域占据主导地位，但受限于推理开销。虽然模型量化缓解了边缘部署的这些瓶颈，但由于两个关键挑战，静态量化方法对于 VLA 而言仍非最优：(1) 时间动态敏感性，即固定精度忽略了随阶段变化的误差容忍度，从而浪费了资源；(2) 实时分配，即如何识别实时敏感性以指导位宽分配仍是一个未解决的问题。为了应对这些挑战，我们提出了 DyQ-VLA，一种针对 VLA 的动态量化框架。具体而言，一种敏感性感知切换策略利用实时运动学代理来触发位宽切换，而运动学引导模块则动态分配最优位宽。实验表明，DyQ-VLA 仅需原始内存占用的 30.9%，同时保持了 99.5% 的原始性能，实现了 1.49 倍的仿真加速和高达 1.43 倍的真实世界加速。

## Abstract
Vision-Language-Action (VLA) models are dominant in embodied intelligence but are constrained by inference overheads. While model quantization alleviates these bottlenecks for edge deployment, static quantization approaches remain suboptimal for VLAs due to two critical challenges: (1) Temporal-dynamic sensitivity, where fixed precision wastes resources by ignoring stage-varying error tolerances; and (2) Real-time allocation, where identifying real-time sensitivity to guide bit allocation remains unsolved. To address these challenges, we propose DyQ-VLA, a dynamic quantization framework for VLAs. Specifically, a sensitivity-aware switching strategy leverages real-time kinematic proxies to trigger the bit-width switch, while a kinematic-guided module dynamically allocates the optimal bit-width. Experiments show that DyQ-VLA requires only 30.9% of the original memory footprint while maintaining 99.5% of its original performance, achieving 1.49x simulation and up to 1.43x real-world speedups.

### 1. 核心问题与整体含义（研究动机和背景）
视觉-语言-动作（VLA）模型虽然在具身智能中表现卓越，但其巨大的计算和内存开销限制了在边缘设备上的实时部署。传统的**静态量化**方法在处理 VLA 时面临两大挑战：
*   **时间动态敏感性（Temporal-dynamic sensitivity）：** VLA 模型在不同执行阶段对误差的容忍度不同。在粗粒度移动时，微小的量化误差无关紧要；但在精细操作（如抓取、插入）时，同样的误差会导致任务失败。静态量化为了保证成功率必须全程维持高精度，造成了资源浪费。
*   **实时分配难题（Real-time allocation）：** 缺乏一种轻量级的实时指标来识别当前的量化敏感度，从而无法在推理过程中动态调整位宽。

### 2. 方法论：核心思想与关键技术
DyQ-VLA 提出了一个**动态量化框架**，核心思想是利用机器人的**运动学状态（Kinematics）**作为量化敏感度的实时代理。

*   **核心组件：**
    1.  **敏感度感知切换策略（Sensitivity-Aware Switching Strategy）：** 采用“静态权重-动态激活”（W4AX）模式。权重固定为 4-bit 以节省显存，激活值则根据敏感度在 BF16、8-bit、4-bit、2-bit 之间动态切换。
    2.  **运动学引导位宽分配模块（Kinematic-Guided Bit Allocation Module）：** 
        *   **特征融合：** 提取“运动精细度”（$M_t$，反映平移幅度）和“角抖动”（$J_t$，反映旋转波动）。$M_t$ 捕捉宏观趋势，$J_t$ 捕捉微观突变。
        *   **滞后切换机制：** 引入非对称滞后算子和延迟窗口，防止位宽在阈值边缘频繁震荡，确保系统吞吐量。
        *   **离线校准：** 通过离线分析确定不同敏感度下的误差边界，生成实时查找表。
*   **系统实现：** 采用异步 CPU-GPU 流水线。CPU 在 GPU 进行视觉预处理时计算运动学指标并决定位宽，通过零拷贝（Zero-Copy）内存传递标志位，实现零调度开销。
