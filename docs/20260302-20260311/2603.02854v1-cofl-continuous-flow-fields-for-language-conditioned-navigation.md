## 摘要
语言条件导航流水线通常依赖于脆弱的模块化组件或成本高昂的动作序列生成。为了解决这些局限性，我们提出了 CoFL，这是一种端到端策略，可将鸟瞰图 (BEV) 观测和语言指令直接映射为用于导航的连续流场。CoFL 并不预测离散的动作标记，也不通过迭代去噪采样动作块，而是输出可在任意 2D 投影位置查询的瞬时速度。轨迹通过对预测场进行数值积分获得，从而产生在闭环执行下保持反应性的平滑运动。为了实现大规模训练，我们构建了一个包含超过 50 万个 BEV 图像-指令对的数据集，每个数据对都通过程序化方式标注了流场和轨迹，这些流场和轨迹源自基于 Matterport3D 和 ScanNet 构建的 BEV 语义图。通过在混合分布上进行训练，CoFL 在严格未见过的场景中显著优于基于模块化视觉语言模型 (VLM) 的规划器和生成式策略基准。最后，我们在具有多种布局的顶视 BEV 观测的真实世界实验中零样本部署了 CoFL，保持了可靠的闭环控制和高成功率。

## Abstract
Language-conditioned navigation pipelines often rely on brittle modular components or costly action-sequence generation. To address these limitations, we present CoFL, an end-to-end policy that directly maps a bird's-eye view (BEV) observation and a language instruction to a continuous flow field for navigation. Instead of predicting discrete action tokens or sampling action chunks via iterative denoising, CoFL outputs instantaneous velocities that can be queried at arbitrary 2D projected locations. Trajectories are obtained by numerical integration of the predicted field, producing smooth motion that remains reactive under closed-loop execution. To enable large-scale training, we build a dataset of over 500k BEV image-instruction pairs, each procedurally annotated with a flow field and a trajectory derived from BEV semantic maps built on Matterport3D and ScanNet. By training on a mixed distribution, CoFL significantly outperforms modular Vision-Language Model (VLM)-based planners and generative policy baselines on strictly unseen scenes. Finally, we deploy CoFL zero-shot in real-world experiments with overhead BEV observations across multiple layouts, maintaining reliable closed-loop control and a high success rate.

# 论文总结：CoFL (Continuous Flow Fields for Language-Conditioned Navigation)

## 1. 核心问题与整体含义
*   **研究动机：** 传统的语言条件导航方法通常分为两类：一是**模块化流水线**（感知-定位-规划），这类方法容易产生上游误差累积且系统脆弱；二是**端到端策略**，如基于 Token 的动作预测（离散、不平滑）或基于扩散模型（Diffusion Policy）的动作块生成（推理延迟高、缺乏显式几何表示）。
*   **核心问题：** 如何在保持端到端简洁性的同时，生成几何感知强、运动平滑且能实时响应的导航轨迹？
*   **整体含义：** 论文提出了 **CoFL**，将导航任务重新表述为**连续流场（Continuous Flow Field）估计**。它直接将鸟瞰图（BEV）观测和语言指令映射为一个空间向量场，通过对该场进行数值积分来生成平滑且避障的导航路径。

## 2. 方法论
*   **核心思想：** 学习一个参数化的流场函数 $v_\theta(x | I, \ell)$，其中 $x$ 是空间坐标，$I$ 是 BEV 图像，$\ell$ 是语言指令。该函数为空间中任意位置提供一个指向目标的瞬时速度向量。
*   **关键技术细节：**
    *   **架构：** 采用 Transformer 结构。编码器使用预训练的 **SigLIP 2** 提取视觉和文本特征并进行融合；解码器通过查询（Query）机制，输入 2D 坐标（经高斯傅里叶编码），输出对应位置的速度幅度和方向。
    *   **流场预测：** 解码器不直接回归速度，而是分别预测**非负幅度**（通过 Softplus）和**单位方向**，以增强数值稳定性。
    *   **轨迹生成：** 在推理时，从机器人当前位置出发，利用**前向欧拉积分（Forward Euler Integration）**在预测的流场中进行迭代，生成连续轨迹。
    *   **时间缩放：** 引入了逆时间缩放机制，使得机器人能以接近恒定的速度向目标移动，并在接近目标时平稳停止。
*   **算法流程：** 
    1. 提取 BEV 和指令特征。
    2. 在空间网格上查询流场。
    3. 从起始点开始，根据流场速度进行欧拉积分 rollout。
    4. 执行生成的平滑轨迹。

## 3. 实验设计
*   **数据集：** 构建了一个大规模数据集，包含来自 **Matterport3D** 和 **ScanNet** 的 **50 万+** 个样本。每个样本包含 BEV 图像、多模板合成的语言指令、以及程序化生成的真值流场和轨迹。
*   **Benchmark：** 采用严格的**场景级划分（Scene-wise split）**，确保测试场景在训练集中从未出现，验证模型的泛化能力。
*   **对比方法：**
    *   **Pure VLM:** 使用 Gemini-2.5-Flash 直接预测路径点。
    *   **VLM + Planner:** VLM 识别目标和障碍物，结合 A* 算法规划。
    *   **Diffusion Policy (DP) 系列:** 包括基于 CNN 和 Transformer 的扩散模型变体（DDPM、Flow Matching）。
*   **评估指标：** 终点误差 (FGE)、碰撞率 (CR)、路径曲率 (Curv，衡量平滑度)、路径长度比 (PLR) 以及流场角度/幅度误差。

## 4. 资源与算力
*   **训练算力：** 使用单张 **NVIDIA RTX 4090** GPU 进行训练。
*   **训练配置：** 训练 50 个 Epoch，Batch Size 为 32，每步采样 1000 个空间点进行监督。
*   **推理算力：** 真实世界部署使用笔记本电脑（Intel i9-13900H + **RTX 4090 Laptop GPU**），在 100x100 的查询网格下，端到端延迟约为 **28.57ms**。

## 5. 实验数量与充分性
*   **实验规模：** 进行了大规模的仿真验证（覆盖数千个测试样本）和消融实验。
*   **消融实验：** 针对编码器深度、解码器深度、隐藏维度、自注意力机制、以及时间缩放策略进行了详细对比。
*   **充分性与公平性：** 实验设计较为客观。为了公平对比，所有 Diffusion Policy 基线都使用了与 CoFL 相同的视觉语言编码器。通过“伪退避视界（Pseudo Receding-Horizon）”协议验证了 DP 的性能瓶颈并非源于开环执行，增强了结论的说服力。

## 6. 主要结论与发现
*   **安全性与精度：** CoFL 在未见场景中的碰撞率显著低于所有基线（在 Matterport3D 上 CR 仅为 0.17-0.22，而 DP 基线高达 0.68-0.86），同时保持了极高的目标到达精度。
*   **平滑性：** 相比于路径点预测，流场积分生成的轨迹在曲率上表现更优，运动更自然。
*   **零样本迁移：** 仅在仿真数据上训练的模型，无需任何微调即可直接部署在真实的差分驱动机器人上，成功处理了复杂的空间关系指令（如“移动到椅子的左侧”）。

## 7. 优点
*   **表示法创新：** 将导航建模为连续向量场，解决了离散动作不平滑和扩散模型推理慢的痛点。
*   **高效性：** 单次前向传播即可获得全局一致的引导，支持实时闭环控制。
*   **鲁棒性：** 显式的几何感知（通过流场监督）使其在避障能力上远超纯生成式模型。
*   **开源贡献：** 提供了大规模、高质量的 BEV 导航数据集生成流水线。

## 8. 不足与局限
*   **BEV 依赖：** 模型性能高度依赖于 BEV 观测的质量。在缺乏顶视相机或无人机支援的场景下，如何从车载相机实时构建高质量 BEV 仍是挑战。
*   **安全边际固定：** 训练数据中的避障距离（Clearance）是预设的，模型无法根据用户需求动态调整“激进”或“保守”的驾驶风格。
*   **动态环境：** 虽然支持闭环重规划，但流场预测基于静态图像，对于高速移动障碍物的预测和规避能力尚待进一步验证。
*   **维度局限：** 目前仅限于 2D 平面导航，尚未扩展到 3D 空间（如无人机避障）或更复杂的操纵任务。
