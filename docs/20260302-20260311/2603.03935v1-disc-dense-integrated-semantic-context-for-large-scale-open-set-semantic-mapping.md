## 摘要
开集语义地图构建实现了语言驱动的机器人感知，但目前的以实例为中心的方法受限于缺乏上下文且计算昂贵的基于裁剪（crop-based）的特征提取。为了克服这一根本限制，我们引入了 DISC（Dense Integrated Semantic Context），其特点是采用了一种新颖的单次传递、距离加权的提取机制。通过直接从视觉 Transformer 的中间层导出高保真 CLIP 嵌入，我们的方法消除了传统图像裁剪带来的延迟和领域偏移伪影，从而产生纯净的、掩码对齐的语义表示。为了在大规模连续建图中充分利用这些特征，DISC 构建在全 GPU 加速的架构之上，用精确的实时体素级实例细化取代了周期性的离线处理。我们在标准基准数据集（Replica、ScanNet）以及基于 Habitat-Matterport 3D (HM3DSEM) 新生成的大规模建图数据集上评估了我们的方法，以评估其在多层建筑复杂场景中的可扩展性。广泛的评估表明，DISC 在语义准确性和查询检索方面均显著超越了当前的先进零样本（zero-shot）方法，为机器人部署提供了一个鲁棒且具备实时能力的框架。完整的源代码、数据生成和评估流程将在 https://github.com/DFKI-NI/DISC 上公开。

## Abstract
Open-set semantic mapping enables language-driven robotic perception, but current instance-centric approaches are bottlenecked by context-depriving and computationally expensive crop-based feature extraction. To overcome this fundamental limitation, we introduce DISC (Dense Integrated Semantic Context), featuring a novel single-pass, distance-weighted extraction mechanism. By deriving high-fidelity CLIP embeddings directly from the vision transformer's intermediate layers, our approach eliminates the latency and domain-shift artifacts of traditional image cropping, yielding pure, mask-aligned semantic representations. To fully leverage these features in large-scale continuous mapping, DISC is built upon a fully GPU-accelerated architecture that replaces periodic offline processing with precise, on-the-fly voxel-level instance refinement. We evaluate our approach on standard benchmarks (Replica, ScanNet) and a newly generated large-scale-mapping dataset based on Habitat-Matterport 3D (HM3DSEM) to assess scalability across complex scenes in multi-story buildings. Extensive evaluations demonstrate that DISC significantly surpasses current state-of-the-art zero-shot methods in both semantic accuracy and query retrieval, providing a robust, real-time capable framework for robotic deployment. The full source code, data generation and evaluation pipelines will be made available at https://github.com/DFKI-NI/DISC.

### 1. 核心问题与整体含义
*   **研究动机**：现有的开集（Open-set）语义地图构建方法主要面临两大瓶颈：
    1.  **特征提取缺陷**：主流方法依赖于对图像进行实例裁剪（Crop-based）再送入 CLIP 编码。这不仅计算开销巨大，且由于 CLIP 是在完整图像上预训练的，裁剪会导致严重的“领域偏移”和“全局上下文丢失”，降低识别准确度。
    2.  **可扩展性不足**：在大规模场景（如多层建筑）中，现有的实例细化算法通常依赖 CPU 端的启发式搜索或昂贵的离线后处理，难以满足机器人实时在线建图的需求。
*   **整体含义**：DISC 旨在通过全 GPU 加速的架构和一种无需裁剪的单次传递特征提取机制，实现大规模、高保真、实时的开集语义地图构建。

### 2. 方法论
*   **核心思想**：将整个建图流程（分割、跟踪、特征提取、实例融合）全部迁移至 GPU，并利用 Vision Transformer (ViT) 的中间层特征直接生成语义嵌入。
*   **关键技术细节**：
    *   **单次传递特征提取（Single-pass Extraction）**：受 MaskCLIP 启发，DISC 直接从 CLIP 的倒数第二层 Transformer 提取 patch 级别的特征。
    *   **距离加权显著性图（Spatial Distinctiveness Map）**：为了避免背景噪声干扰，提出了一种计算 patch 特征与空间均值之间残差范数的机制，为纹理丰富、具有显著语义信息的区域分配更高权重。
    *   **实时体素级细化**：利用 GPU 加速的稀疏矩阵算法和边界体积层次结构（BVH），在每一帧实时计算 3D 体素重叠和 DINOv2 视觉相似度，从而在线合并过分割的实例，无需离线后处理。
    *   **视图质量融合（View-quality Fusion）**：定义了一个综合得分 $Q$，结合了几何质量（物体大小、观测角度）、语义一致性（局部与全局特征匹配度）和距离显著性，确保只有高质量的观测会更新地图特征。

### 6. 主要结论与发现
*   **性能领先**：DISC 在所有主要指标上均优于现有的零样本方法，甚至在某些指标上接近或超过了使用监督学习模型的 OpenFusion。

### 7. 优点
*   **高效性**：通过单次推理获取全图特征，极大降低了计算延迟。
*   **鲁棒性**：引入视图质量评分机制，有效防止了错误合并和特征稀释。
*   **工程完备性**：提供了从数据生成、实时建图到评估的完整流水线，并开源了代码。
*   **大规模适配**：解决了开集语义地图在大型复杂环境（多层建筑）中难以实时运行的痛点。

### 8. 不足与局限
*   **依赖前端分割**：系统的上限受限于 2D 分割模型（如 FastSAM）的质量，若 2D 分割出现欠分割或漏检，会直接影响 3D 地图。

*   **静态假设**：目前的体素集成机制隐含地假设环境是静态的，处理动态障碍物的能力有限。
