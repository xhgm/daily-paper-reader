# Context Channel Capacity: An Information-Theoretic Framework for Understanding Catastrophic Forgetting 

# Ran Cheng March 10, 2026 

Abstract 

Catastrophic forgetting remains a central challenge in continual learning (CL), yet lacks a unified information-theoretic explanation for why some architectures forget catastrophically while others do not. We introduce Context Channel Capacity (Cctx ), the mutual information between a CL architecture’s context signal and its generated parameters, and prove that zero forgetting requires Cctx ≥ H(T ), where 

H(T ) is the task identity entropy. We establish an Impossibility Triangle —zero forgetting, online learning, and finite parameters cannot be simultaneously satisfied by sequential state-based learners—and show that conditional regeneration architectures (HyperNetworks) bypass this triangle by redefining parameters as function values rather than states. We validate this framework across 8 CL methods on Split-MNIST (1,130+ experiments over 86 days, 4 seeds each), showing that Cctx perfectly predicts forgetting behavior: methods with Cctx = 0 

(NaiveSGD, EWC, SI, LwF, CFlow) exhibit catastrophic forgetting (6–97%), while methods with Cctx ≈ 1

(HyperNetwork) achieve zero forgetting (98.8% ACC). We further propose Wrong-Context Probing (P5), a practical diagnostic protocol for measuring Cctx , and extend the framework to CIFAR-10 via a novel 

Gradient Context Encoder that closes the oracle gap from 23.3pp to 0.7pp. A systematic taxonomy of 15+ closed research directions—including the Hebbian null result (frozen random features outperform learned features), CFlow’s θ0-memorizer phenomenon, and the SN symmetry barrier to column specialization— provides the community with precisely diagnosed negative results. Our central design principle: architecture over algorithm —the context pathway must be structurally unbypassable. 

# 1 Introduction 

Catastrophic forgetting—the abrupt loss of previously acquired knowledge when a neural network learns new tasks sequentially—was first documented over three decades ago (McCloskey and Cohen, 1989) and remains a central unsolved problem in machine learning (French, 1999). Despite sustained research effort, the field lacks a principled, quantitative framework that explains why some continual learning (CL) architectures forget catastrophically while others do not. Existing approaches to mitigating forgetting fall into three broad families. Regularization-based methods, such as Elastic Weight Consolidation (EWC) (Kirkpatrick et al., 2017) and Synaptic Intelligence (SI) (Zenke et al., 2017), penalize changes to parameters deemed important for previous tasks. Replay-based 

methods store exemplars from past tasks and interleave them during training, effectively breaking the strict sequential constraint. Architecture-based methods, including Progressive Networks (Rusu et al., 2016) and HyperNetworks (von Oswald et al., 2020; Ehret et al., 2021), allocate or generate task-specific parameters. Each family has produced methods that work to varying degrees, but the performance landscape is strikingly heterogeneous: on Split-MNIST, EWC achieves 18.9% accuracy while a HyperNetwork achieves 98.8%—a gap of 80 percentage points using architectures of comparable parameter count (Table 1). 

The missing explanation. What accounts for this 80-point gap? It is not simply a matter of “more sophisticated algorithms”: EWC uses Fisher information, SI tracks path integrals, and CFlow solves neural ODEs—all arguably more complex than the simple MLP used by the HyperNetwork. Nor is it a matter of capacity: the CFlow system has 2.7M parameters while the HyperNetwork has fewer than 100K. # 2 Theoretical Framework 

We develop an information-theoretic framework for continual learning in five stages. First, we formalize CL as constrained online coding (§2.1). Second, we establish that forgetting is information-theoretically inevitable for sequential state-based learners (§2.2). Third, we prove an impossibility triangle (§2.3). Fourth, we introduce Context Channel Capacity and derive the CCC bound (§2.4). Fifth, we classify CL architectures by their Cctx regime and explain why HyperNetworks bypass the impossibility triangle (§2.5–§2.6).

Connection to rate-distortion theory. The CL problem can be viewed as a constrained online source coding problem. The parameter space Θ is a channel of capacity C ≈ d · log 2(1 /δ ) bits, where δ is the effective parameter precision. Each task Dk is a “source” that must be encoded into this channel, and the distortion is the classification error. The crucial difference from standard rate-distortion theory is the causal constraint :the encoder at time k sees only the current source Dk and the previous codeword θk−1, not the full source sequence (D1, . . . , D K ). # 2.2 The Information Bottleneck Chain 

We now establish that forgetting is information-theoretically inevitable for any sequential learner with finite capacity. 

Theorem 1 (Forgetting is Information-Theoretically Inevitable) . For a deterministic sequential learner with finite parameter capacity C = d · log 2(1 /δ ) bits and independent tasks {Dk}Kk=1 , the mutual information between the final parameters and any past task dataset satisfies: 

I(θK ; D1) ≤ I(θK−1; D1) ≤ · · · ≤ I(θ1; D1) ≤ C. (5) 

Proof. Since θk = U (θk−1, D k) is a deterministic function and Dk is independent of D1 given θk−1 (by the causal constraint and task independence), the random variables form a Markov chain: 

D1 → θ1 → θ2 → · · · → θK . (6) To verify the Markov property: given θk−1, the variable θk = U (θk−1, D k) depends on D1 only through θk−1,since Dk is independent of all previous data. By the Data Processing Inequality (DPI), for any Markov chain 

X → Y → Z, we have I(X; Z) ≤ I(X; Y ). Applying this iteratively: 

I(θK ; D1) ≤ I(θK−1; D1) ≤ · · · ≤ I(θ1; D1). (7) The initial bound follows from the fact that θ1 ∈ Θ can represent at most C bits of information about any quantity, yielding I(θ1; D1) ≤ H(θ1) ≤ C.

Corollary 2 (Forgetting Lower Bound) . If K tasks are mutually independent and each requires at least 

Rmin bits of task-specific information for nontrivial performance (i.e., I(θ; Dk) ≥ Rmin is necessary for 

ACC k(θ) > ACC chance ), then when K > C/R min , it is impossible to retain nontrivial performance on all K

tasks simultaneously. Proof. By the Markov chain, all task-specific information must pass through the bottleneck θK . Since the tasks are independent, the total information required for nontrivial performance on all K tasks is at least 

K · Rmin . But I(θK ; D1, . . . , D K ) ≤ H(θK ) ≤ C. When K · Rmin > C , the pigeonhole principle implies that at least one task must have I(θK ; Dj ) < R min , and hence ACC j (θK ) ≤ ACC chance .

Relation to prior work. Theorem 1 formalizes the intuition that forgetting is a necessary consequence of finite capacity, a point made informally by French (1999) and partially formalized by Taheri and Thrampoulidis (2025) for single-hidden-layer networks. Our contribution is to frame this as a consequence of the DPI applied to a Markov chain, which (i) requires no assumptions on network architecture or training procedure, and (ii) makes the role of the causal constraint explicit: it is the sequential structure D1 → θ1 → · · · → θK that creates the information bottleneck. Methods that break this chain (e.g., replay, joint training) are not subject to Eq. (5).

# 2.4 Context Channel Capacity 

The impossibility triangle (Theorem 3) applies to sequential state-based learners —systems where θk is obtained by modifying θk−1. We now introduce the key concept that distinguishes architectures that bypass this limitation. 

Definition 4 (Context Signal) . A context signal for a CL architecture is any input c ∈ C that carries information about the current task identity. Examples include: oracle task ID (one-hot vector), batch statistics (mean, variance of current inputs), gradient signals, or learned task embeddings. For architectures without an explicit context mechanism (e.g., SGD, EWC, SI), we define c = ∅ (no context signal).

Intuition. Cctx measures how many bits of task-identifying information can flow through the architecture’s context pathway to influence the prediction parameters. If Cctx = 0 , the architecture has no mechanism to produce different parameters for different tasks—and therefore must use a single set of parameters for all tasks, leading to interference. If Cctx ≥ H(T ), the architecture can, in principle, produce fully distinct parameters for each task, eliminating interference entirely. We now state our main theoretical result.

2.5.2 Paradigm B: State Transformation ( Cctx → 0)

In this paradigm, a context signal ck exists and is combined with the parameter state to produce task-adapted parameters: 

θk = f (θk−1, c k), (24) where f is a learned transformation (e.g., a neural ODE in CFlow). The key distinction from Paradigm C is that f takes both the previous state θk−1 and the context ck as inputs. In principle, this allows context to modulate the transformation. In practice, the following structural failure mode arises. 

Remark 2 (Dimensionality mismatch and context bypass) . When dim (θ) ≫ dim (c) and f is parameterized as a neural network operating on the concatenation [θ; c], the optimizer generically learns to encode task information in θ (the high-dimensional input) rather than in c (the low-dimensional input). This is because the gradient magnitude scales with input dimension: ∥∂f /∂θ ∥ ∝ √d while ∥∂f /∂c ∥ ∝ √m where m = dim (c) ≪

d = dim( θ).

Our CFlow system exemplifies this failure mode. CFlow uses a neural ODE where the flow network takes as input the concatenation [θ ∈ R4842 ; c ∈ R32 ]—a dimensionality ratio of ∼ 150 : 1. Despite the presence of an explicit context encoder, CFlow exhibits ∆P5 = 0 .0, confirming that the context signal is structurally invisible to the dynamics. The 92.4% accuracy of CFlow comes entirely from the meta-learned initialization 

θ0, not from context-dependent parameter generation. In the language of Cctx :

CCFlow ctx ≈ 0 (structurally, despite c̸ = ∅). (25) This is a concrete example of the distinction between having a context signal and using it. The Cctx framework correctly classifies CFlow as effectively context-free, despite its architectural appearance.

Formal analysis. Let us compare the information flows: Paradigm A: D1 → θ1 → · · · → θK (Markov chain, DPI applies) (29) Paradigm C: {D1, . . . , D K } → ϕ∗ → θk = gϕ∗ (ck) (no Markov chain) (30) In Paradigm A, the DPI guarantees monotonic information loss: I(θK ; D1) ≤ I(θ1; D1). The information I(θk; Dk) is not constrained by any sequential bottleneck; it is constrained only by Cctx (how much task-specific information flows through the context) and the expressiveness of g.

The “unbypassability” principle. The above analysis reveals why architectural design matters more than algorithmic sophistication. In Paradigm C, the context pathway is the only route for task information to reach θk. There is no high-dimensional state θk−1 that can serve as an alternative information channel. This “unbypassability” is precisely what forces the model to use context—and thereby achieve high Cctx .Contrast this with CFlow (Paradigm B), where the context c is concatenated with the 4842-dimensional state θ. Here, the state provides a massively wider alternative channel, and the optimizer rationally ignores the narrow context pathway. The result: ∆P5 = 0 .0 (context is unused) despite the architecture nominally having a context input. 

Remark 3 (Connection to the attention–RNN analogy) . The relationship between Paradigm A and Paradigm C mirrors the relationship between recurrent neural networks and attention mechanisms in sequence modeling. An RNN maintains a hidden state ht = f (ht−1, x t) that suffers from vanishing gradients—the sequential analog of catastrophic forgetting. Attention mechanisms bypass the sequential bottleneck by directly accessing past inputs via content-based addressing. Similarly, HyperNetworks bypass the sequential parameter bottleneck by directly generating task-specific parameters via context-based addressing. In both cases, the solution is to replace sequential state compression with direct conditional generation .

3.1.3 HSPC-T Column Specialization—The SN Impossibility Mechanism. The Hierarchical Sparse Predictive Coding with Temporal pressure (HSPC-T) hypothesis proposed that temporal pressure (urgency signals, metabolic costs) could drive individual columns in a dictionary learning network to specialize for different tasks, creating implicit context through column routing.

The SN symmetry barrier. All three sub-directions failed for the same fundamental reason: 

Proposition 9 (SN Permutation Symmetry of Reconstruction Loss) . For a dictionary learning network with columns {(wi, e i)}Ni=1 minimizing reconstruction loss L = ∥x − Wdz∥2 where z = encode (x; We), the loss is invariant under the symmetric group SN acting by permutation of column indices:

Consequently, the gradient field ∇wi L is identical in structure for all columns i, and no loss-driven mechanism can break this symmetry to induce task specialization.

Cctx interpretation. Column specialization was an attempt to create emergent context through the network’s internal dynamics. The SN symmetry result shows this is impossible without explicit symmetry-breaking—which is precisely what an explicit context pathway provides.

# 4 Wrong-Context Probing Protocol 

We formalize the Wrong-Context Probing (P5) protocol introduced in Section 2.4 and extend it to a family of diagnostic probes that collectively characterize how a CL architecture uses (or fails to use) context information. 

# 4.1 Protocol Definition 

Definition 6 (P5 Probing Family) . For a context-conditional CL model with context encoder e, parameter generator g, and base parameters θbase , we define the following probes: 

P5a : Wrong task identity (for oracle context). Evaluate task k using context c(k+1) mod K . Measures: whether oracle task ID is actually used. 

P5b : Random context vector . Evaluate task k using c ∼ N (0 , I ). Measures: whether any context information (not just correct) matters. 

P6 : Random base parameters . Replace θbase with random initialization, keep correct context. Measures: dependence on meta-learned initialization. 

P7 : Zero context . Evaluate with c = 0. Measures: baseline performance of θbase alone. For each probe Pi, we report ∆Pi = ACC Pi − ACC normal .

14 Interpretation grid. The combination of probe results reveals the architecture’s information routing: 

• ∆P5 ≈ 0, ∆P6 ≪ 0: “θ0 memorizer” (CFlow pattern). Context is dead; all information is in the initialization. 

• ∆P5 ≪ 0, ∆P6 ≈ 0: “Context-dependent” (HyperNet pattern). Performance comes entirely from context-conditional parameter generation. 

• ∆P5 ≈ 0, ∆P6 ≈ 0: Context and initialization both irrelevant (degenerate case).

# 7.1 Architecture > Algorithm: The Cctx -First Design Principle 

The central lesson of our 1,130+ experiments is that the topology of information flow matters far more than the algorithm operating within that topology. EWC, SI, and LwF all implement sophisticated regularization algorithms, yet achieve ≤ 24 .2% ACC on Split-MNIST—barely above chance for the later tasks. Meanwhile, a HyperNetwork with a trivially simple context encoder (one-hot task identity passed through a single MLP) achieves 98.8% ACC with zero forgetting. The difference is entirely structural. We distill this into three concrete design principles for CL architectures: 1. Explicit context signal. The architecture must have a well-defined input that carries task-identifying information—whether task identity, batch statistics, or gradient signatures. Without this signal, 

Cctx = 0 by definition and no algorithm can prevent forgetting (Corollary 2). 2. Structural unbypassability. All task-specific computation must flow through the context pathway. There must be no parameter pathway that can encode task information without context. Formally, for parameter generator θ(c) = g(c, ϕ ) where ϕ denotes context-independent parameters, we require that 

I(θ; T | c) = 0 —the parameters carry no task information beyond what context provides. 3. Differentiable context encoding. The mapping from raw observations to context vectors must be end-to-end differentiable with respect to the encoder parameters. This is not merely an engineering convenience: our learned encoder experiments revealed that EMA-based running statistics ( ¯μk, ¯σ2

> k

)use in-place buffer operations that are not tracked by autograd , causing the encoder to collapse to a near-constant mapping ( ∆P5 = 0 ). Replacing EMA with direct batch statistics ( direct_stats=True )restored ∆P5 from 0.0 to −92 .2 percentage points on Split-MNIST. CFlow (NeuralDrift Team, 2026) provides a vivid illustration of what happens when Principle 2 is violated. Its architecture concatenates a 32-dimensional context vector with a 4,842-dimensional parameter vector θ to form the input to an ODE flow network. Despite the context encoder being well-defined (Principle 1) and differentiable (Principle 3), the 150 : 1 dimensionality ratio between θ and c creates a structural bypass: the optimizer finds it energetically cheaper to encode all task information in the meta-learned initialization θ0

than to route gradients through the thin context pathway. The result is ∆P5 = 0 .0—the model completely ignores its context signal—and all performance (ACC = 92 .4% ) comes from the memorized θ0.

Remark 4. The Cctx -first principle inverts the typical CL research workflow. Instead of asking “given a fixed architecture, what algorithm prevents forgetting?”, we argue one should ask “given the need for Cctx ≥ H(T ),what architecture ensures the context pathway is unbypassable?” The algorithm is secondary. 

# 7.2 The Learned Context Encoder Problem 

A critical question for practical deployment is: can the context signal be learned from data rather than provided by an oracle? Our experiments reveal that the answer depends sharply on the benchmark. 

MNIST: batch statistics suffice. On Split-MNIST, the 5 tasks involve distinct digit pairs (0/1, 2/3, . . . , 8/9) whose pixel-level statistics differ substantially. The pairwise cosine similarity between batch mean vectors across tasks is approximately 0.5, providing ample signal for task discrimination. The learned encoder with direct batch statistics achieves 99.1% ACC ( ∆P5 = −95 .2), actually exceeding the oracle encoder (98.8%). 

CIFAR-10: batch statistics collapse. On Split-CIFAR-10 (5 tasks, 2 classes each), all natural image classes share similar low-level statistics. The pairwise cosine similarity between batch means exceeds 0.995, and the learned batch-statistics encoder achieves only 54.4% ACC—equivalent to memorizing the final task. 

Solution: Gradient Context Encoder. To address this, we propose using the loss gradient ∇θbase L as the context signal. The key insight is that gradients are inherently task-diagnostic: different tasks induce different update directions on shared parameters, even when input statistics are indistinguishable. Concretely, we compute a mini-batch gradient, project it through a frozen random matrix to R128 , and pass the result through a small MLP to produce a 64-dimensional context vector. 20 The gradient context encoder closes the oracle gap dramatically: Batch statistics: ACC = 54 .4% , ∆oracle = 23 .3pp (47) Gradient context: ACC = 77 .0 ± 0.9% , ∆oracle = 0 .7pp (48) Analysis of inter-task gradient cosine similarities confirms the mechanism: gradients computed with real labels yield pairwise cosine ≈ − 0.19 (near-orthogonal across tasks), while pseudo-label gradients yield cosine 

≈ 0.95 (task-indistinguishable). The gradient context encoder’s effectiveness thus depends critically on access to real labels during context computation. 

Limitation. The gradient context encoder requires labeled samples at inference time to compute ∇θbase L.In a deployment scenario, this means the system needs a small labeled calibration set per task—analogous to a “task prompt”—before it can generate task-appropriate parameters. While this is acceptable in many practical settings (e.g., few-shot calibration), it prevents fully unsupervised deployment.

# 7.4 Implications for CL Research 

The value of negative results. Our experimental campaign produced 15+ closed research directions, each documented with a precise mathematical or empirical root cause. These are not failures of hyperparameter tuning; they are structural impossibilities within the given architecture class. For example, Hebbian learning contributes 0.0 percentage points to DND’s CL performance because a frozen random projection achieves identical accuracy—the Hebbian rule simplifies to an online PCA that discards precisely the high-frequency components needed for task discrimination. We believe the CL community would benefit from systematic 21 publication of such negative results, as they narrow the search space far more efficiently than positive results expand it. 

Biological inspiration requires biological scale. Several of our closed directions—Hebbian learning, metabolic pruning, synaptic consolidation—were inspired by neuroscience. In the brain, these mechanisms operate across 10 11 neurons with complex neuromodulatory control (dopamine, acetylcholine, norepinephrine). When reduced to a two-layer MLP with Oja’s rule, Hebbian learning degenerates to PCA—a linear dimen-sionality reduction that is neither task-specific nor competitive with random projections. The lesson is not that biological mechanisms are irrelevant, but that they require biological-scale complexity to be effective. 

Split-MNIST is a dangerous benchmark. Our results reveal that Split-MNIST has such large inter-task differences that almost any context signal suffices: batch pixel means alone achieve 99.1% ACC. This means Split-MNIST cannot distinguish between architectures with high Cctx and those with low but nonzero Cctx . On CIFAR-10, where inter-task differences are subtle, the gap between oracle (77.7%) and naïve learned encoders (54.4%) is 23.3 percentage points—a failure mode completely invisible on MNIST. We recommend that CL papers always include at least one “hard context” benchmark where batch statistics are uninformative. 

Evaluation methodology. Two methodological findings deserve attention: (i) Quick mode evaluation (reduced epochs and samples) produces a 100% false positive rate in our CL experiments—every method that appeared to work in quick mode failed at full scale. We recommend against using abbreviated training protocols for CL evaluation. (ii) Wrong-Context Probing (P5) should become a standard evaluation tool for any context-conditional CL architecture, as it provides a direct, interpretable measure of whether the architecture actually uses its context signal. 

# 7.5 Limitations 

Benchmark scope. Our systematic validation is conducted on Split-MNIST (5 tasks) with extension experiments on Split-CIFAR-10. Validating Cctx predictions on harder benchmarks—Split-CIFAR-100 (20 tasks), Split-ImageNet, and domain-incremental settings—remains important future work. The key question is whether the sharp Cctx = 0 ⇔ forgetting dichotomy persists when the number of tasks grows and inter-task similarity increases. 

Task-incremental assumption. The Cctx framework assumes that a context signal carrying task-identifying information is available or can be inferred. In task-free CL settings—where task boundaries are not provided and the data distribution shifts gradually—the notion of “context” is less well-defined. Extending the framework to continuous distribution shifts, where T is a continuous random variable rather than a discrete task index, is nontrivial. 

P5 as a binary probe. The Wrong-Context Probing protocol provides a binary signal: either the model uses context ( ∆P5 ≪ 0) or it does not ( ∆P5 ≈ 0). It does not provide a continuous, calibrated estimate of Cctx in bits. A method with ∆P5 = −95% and one with ∆P5 = −50% both “use context,” but differ in how much task information the context pathway carries. Developing continuous estimators (e.g., via MINE (Belghazi et al., 2018) or NWJ bounds) would strengthen the framework. 

Task boundary knowledge. The HyperNetwork architecture requires knowledge of task boundaries during training (to accumulate per-task context statistics and apply contrastive losses). While this is a standard assumption in task-incremental CL, it limits applicability to settings where task transitions are identifiable. 

Label requirement for gradient context. The gradient context encoder requires real labels to compute discriminative gradients. Pseudo-label gradients (cosine ≈ 0.95 across tasks) are uninformative. This limits fully unsupervised deployment, though few-shot calibration is feasible in many practical settings. 

# 7.6 Future Directions 

Scaling to harder benchmarks. The immediate priority is validating the Cctx framework on Split-CIFAR-100 (20 tasks, 5 classes each) and Split-ImageNet, where H(T ) = log 2 20 ≈ 4.3 bits demands higher-capacity context channels. The gradient context encoder is a natural candidate, as gradient-based task discrimination should scale with model capacity. 22 Task-free CL via automatic context generation. The NestedCapsule architecture (Section 6.5) provides a promising direction: its routing mechanism produces emergent task specialization (gate entropy dropping from log 2 8 = 3 .0 to ∼ 0.5, direction cosine ≈ − 0.09 ) without explicit task labels. This suggests a path toward task-free CL where context is generated automatically from input statistics and routing dynamics. 

Continuous Cctx estimation. Replacing the binary P5 probe with a continuous estimator based on mutual information neural estimation (Belghazi et al., 2018) or the Nguyen-Wainwright-Jordan bound would enable quantitative comparison of architectures that all “use context” but to different degrees. This is particularly important for comparing HyperNet variants (oracle vs. learned vs. gradient). 

Connection to PAC-Bayes bounds. The Cctx framework has a natural connection to PAC-Bayes analysis of CL (Doan et al., 2021): the context channel capacity plays the role of a complexity term that controls the generalization-forgetting trade-off. Formalizing this connection could yield tighter bounds that account for the specific structure of context-conditional architectures. 

Multi-modal and compositional context. Real-world continual learning often involves multiple simultaneous context signals (task instructions, environmental cues, user identity). Extending Cctx to multi-channel settings, where Cctx = I(c1, c 2, . . . , c m; θ), could provide design guidance for compositional CL systems. 

# 8 Related Work 

# 8.1 Catastrophic Forgetting 

The catastrophic forgetting problem was first identified by McCloskey and Cohen (1989) in connectionist models and systematically surveyed by French (1999). The modern era of CL research was catalyzed by Elastic Weight Consolidation (EWC) (Kirkpatrick et al., 2017), which uses the Fisher information matrix to penalize changes to parameters important for past tasks. Synaptic Intelligence (SI) (Zenke et al., 2017) computes importance online during training, avoiding the need for a separate Fisher computation. Learning without Forgetting (LwF) (Li and Hoiem, 2017) uses knowledge distillation from previous model outputs, requiring no stored data. Experience Replay and its variants (Chaudhry et al., 2019) store and revisit subsets of past data. From the Cctx perspective, all of the above are Cctx = 0 methods: they operate on shared parameters without any mechanism for task-conditional computation. Our framework predicts—and our experiments confirm—that such methods are fundamentally limited, regardless of the sophistication of their regularization or replay strategies. 

# 8.2 Architecture-Based Continual Learning 

Progressive Networks (Rusu et al., 2016) allocate new network columns per task, achieving zero forgetting at the cost of linearly growing parameters. PackNet (Mallya and Lazebnik, 2018) assigns parameter subsets to tasks via iterative pruning. Expert Gate (Aljundi et al., 2017) trains task-specific expert networks with a gating mechanism for task selection. These methods implicitly achieve Cctx > 0 by providing task-specific parameter pathways, but they typically require parameter growth proportional to K. The HyperNetwork approach (von Oswald et al., 2020) achieves a more elegant solution: a fixed-size meta-network generates task-specific parameters from compact context vectors, keeping total parameter count independent of K.von Oswald et al. (2020) first demonstrated that task-conditioned HyperNetworks can achieve strong CL performance on Split-MNIST and Permuted-MNIST. Ehret et al. (2021) extended this to sequential task settings with learned task embeddings. Our contribution relative to this line of work is not architectural—we use a standard HyperNetwork—but explanatory : the Cctx framework provides a formal information-theoretic account of why HyperNetworks succeed, identifies the precise structural properties that are necessary (unbypassability, differentiability), and predicts failure modes (CFlow’s context bypass) that are invisible without this framework. 23 8.3 Information-Theoretic Perspectives on CL 

Several prior works have applied information theory to continual learning, but none analyze the role of context-conditional architectures. Achille and Soatto (2018) establish that the mutual information I(W ; S) between weights and training data controls generalization via a PAC-Bayes-like bound. However, their analysis is restricted to single-task learning and does not address the sequential multi-task setting where forgetting arises. Doan et al. (2021) analyze catastrophic forgetting through the Neural Tangent Kernel (NTK) overlap matrix and derive PAC-Bayes bounds for CL. Their analysis characterizes forgetting in terms of task gradient similarity but does not consider architectures where parameters are conditionally generated from context. Taheri and Thrampoulidis (2025) derive forgetting rate bounds for overparameterized single-hidden-layer networks, showing that forgetting scales with the ratio of task-specific to shared features. Their bounds apply to Cctx = 0 architectures and do not account for conditional parameter generation. BEDS (Caraffa, 2026) argues that forgetting is structurally necessary under Bayesian estimation of distributional shift, providing Fisher-Rao optimal regularization. While philosophically aligned with our impossibility result (Theorem 3), BEDS provides a macroscopic framework without the architectural analysis or the quantitative Cctx predictions that distinguish our work. The key distinction of our framework is the focus on architectural information flow topology : we analyze not just whether forgetting occurs, but how the specific routing of information through context pathways determines whether it must occur. This shifts the question from “how much forgetting?” to “does the architecture permit zero forgetting at all?” 

# 8.4 Compression and Continual Learning 

The connection between compression and learning has a long history, from minimum description length (MDL) (Rissanen, 1978) to the information bottleneck (Tishby et al., 2000). In the CL context, Bornschein et al. (2022) connect prequential MDL to online learning, showing that compressibility of the data stream relates to learnability. Angelini et al. (2023) provide the most direct application of MDL to CL, using variational density propagation with MDL objectives. Their work demonstrates that MDL-based CL is feasible but does not analyze the role of architectural structure in determining compression efficiency. Finzi et al. (2025) introduce epiplexity as an information measure for computationally bounded agents, extending Shannon information to settings where the decoder has limited computational power. This is relevant to our framework because Cctx implicitly assumes that the parameter generator g can decode context into useful parameters—an assumption that may not hold for computationally constrained generators. Our work differs from the compression-CL literature in that we do not propose a compression-based learning algorithm. Instead, we use information-theoretic analysis to characterize architectural capacity for conditional regeneration —the ability to reconstruct task-specific parameters from compact context signals. 

# 8.5 Meta-Learning and Task Conditioning 

The HyperNetwork approach to CL is closely related to meta-learning (Finn et al., 2017; Hospedales et al., 2022), where a meta-learner produces task-specific models from task descriptors. MAML (Finn et al., 2017) and its variants learn an initialization θ0 from which task-specific parameters can be obtained via a few gradient steps. In our framework, MAML’s inner loop serves as an implicit context encoder: the support set + gradient steps produce a task-conditional parameter update. However, because θ0 persists across tasks and carries most of the model’s capacity, MAML exhibits the same bypass vulnerability as CFlow—the “task information” may reside primarily in θ0 rather than in the adaptation. Task-conditioned architectures in multi-task learning (Strezoski et al., 2019) use FiLM layers (Perez et al., 2018), task-specific batch normalization, or adapter modules to modulate a shared backbone. These approaches achieve Cctx > 0 by design, as the task signal directly modulates intermediate representations. # 9 Conclusion 

We have presented Context Channel Capacity ( Cctx ), an information-theoretic framework that provides a unified explanation for catastrophic forgetting in continual learning. Through the lens of Cctx , the diverse landscape of CL methods—from regularization (EWC, SI) to distillation (LwF) to replay to architecture-based approaches (HyperNetworks)—reduces to a single structural question: how much task-identifying information can flow through the architecture’s context pathway? 

Our framework makes three contributions: 

Theoretical. We prove that zero forgetting requires Cctx ≥ H(T ) (Theorem 4) and that the Impossibility Triangle (Theorem 3) is bypassed by conditional regeneration—reconstructing task-specific parameters from compact context signals rather than maintaining them through sequential updates. These results shift the CL question from “how to prevent parameter interference” to “how to ensure sufficient context bandwidth.” 

Empirical. We validate the framework across 8 CL methods on Split-MNIST (1,130+ experiments, 4 seeds per configuration). The Cctx proxy perfectly predicts forgetting behavior: all methods with Cctx = 0 

exhibit catastrophic forgetting (6–97%), while methods with Cctx ≈ 1 achieve zero forgetting. We further demonstrate that the framework extends to harder benchmarks via the Gradient Context Encoder, which achieves 77.0% ACC on Split-CIFAR-10 (within 0.7pp of oracle), and the NestedCapsule architecture, which pushes this to 78.5% with emergent routing specialization. 

Methodological. Wrong-Context Probing (P5) provides a simple, practical diagnostic tool: feed wrong-task context and measure accuracy degradation. A ∆P5 ≈ 0 definitively diagnoses context blindness, regardless of how sophisticated the architecture appears. We recommend P5 as a standard evaluation tool for any context-conditional CL system. The overarching design principle is: architecture over algorithm, context pathway must be structurally unbypassable . No amount of algorithmic sophistication—whether Fisher regularization, synaptic intelligence, knowledge distillation, or Hebbian learning—can compensate for an architecture where task-identifying information has no pathway to influence computation. Conversely, even a simple architecture with a well-designed context channel can achieve zero forgetting. Looking forward, the most important open questions are: (i) scaling Cctx to settings with many tasks (K ≫ 5) and fine-grained inter-task similarity; (ii) extending the framework to task-free CL, where context must be generated automatically without task boundary knowledge; and (iii) connecting Cctx to PAC-Bayes generalization bounds to obtain end-to-end learning guarantees for context-conditional CL architectures. 25 References

# A.1 Proof of Theorem 1 (Information Bottleneck Chain) 

We provide the complete proof with all conditions made explicit. 

Proof. Consider the sequential learning process θ0 → θ1 → · · · → θK , where θk = U (θk−1, D k) is the deterministic update rule. 

Step 1: Markov chain condition. We claim that D1 → θ1 → θ2 → · · · → θK forms a Markov chain. To see this, note that by the causal constraint of sequential learning, θk = U (θk−1, D k) depends only on θk−1

and Dk. Since Dk is drawn independently of D1 (the tasks are independent), we have: 

P (θk | θk−1, D 1, . . . , D k−1) = P (θk | θk−1) (49) where the conditioning on θk−1 implicitly includes the effect of Dk (which is independent of D1). More precisely, for k ≥ 2:

I(θk; D1 | θk−1) = 0 (50) because θk is a deterministic function of θk−1 and Dk, and Dk ⊥ D1.

Step 2: Data Processing Inequality. The DPI (Cover and Thomas, 2006) states that for any Markov chain X → Y → Z:

I(X; Z) ≤ I(X; Y ) (51) Applied to the chain D1 → θk−1 → θk (which holds for all k ≥ 2): 

I(θk; D1) ≤ I(θk−1; D1) ∀ k ≥ 2 (52) By induction, we obtain the full chain: 

I(θK ; D1) ≤ I(θK−1; D1) ≤ · · · ≤ I(θ1; D1) (53) 27 Step 3: Initial capacity bound. The parameter θ1 = U (θ0, D 1) takes values in Θ ⊆ Rd. If each parameter is represented with precision δ (e.g., floating-point quantization), then |Θ| ≤ (1 /δ )d, and: 

I(θ1; D1) ≤ H(θ1) ≤ log 2 |Θ| ≤ d · log 2(1 /δ ) ≜ C (54) 

Discussion: Parameter precision δ. For IEEE 754 float32, the mantissa has 23 bits, so the effective precision per parameter is approximately log 2(1 /δ ) ≈ 24 bits (accounting for the sign bit and distinguishable mantissa states). In practice, the effective precision is often much lower due to the loss landscape geometry: most directions in parameter space are “flat” and carry no task-relevant information. The bound C = d · 24 

bits is therefore a loose upper bound for modern networks with d ∼ 10 6–10 9.

Discussion: Deterministic vs. stochastic update rules. For deterministic U (e.g., full-batch gradient descent), the Markov chain is deterministic, and DPI holds trivially. For stochastic U (e.g., SGD with mini-batch noise), θk is a stochastic function of (θk−1, D k), but the Markov property still holds: I(θk; D1 | θk−1) = 0 

because the stochasticity comes from Dk (independent of D1) and/or the random seed, not from D1. However, stochastic updates can only decrease I(θk; D1) further (adding noise cannot increase mutual information), so the bound remains valid: 

I(θk; D1) ≤ I(θk−1; D1) − I(θk; D1 | θk−1) ≤ I(θk−1; D1) (55) where the second term accounts for the information lost through stochastic processing.

# B.2 CFlow: Continuous-Time Parameter Flow 

CFlow is our ODE-based approach to continual learning, developed as a hypothesis that continuous-time parameter dynamics could enable smooth task transitions. 

Architecture. 

• Task network: ConvTaskNet with 4 filters, producing θdim = 4842 parameters. 

• Flow network: A neural ODE f (θ, c, t ) that generates dθ/dt . Input dimension: θdim + cdim + tembed .

• Context encoder: Learned context encoder mapping batch statistics to c ∈ R32 .

• Time encoding: 16 Fourier frequencies for sinusoidal time embeddings. 

Training. 

• Meta-training over 80 continuous-time episodes with task transitions. 

• Adam optimizer, learning rate 10 −3.

• 30% replay ratio, 100 samples per past task. 

Results and the θ0 memorizer problem. CFlow achieves 92.6% ACC (v1) and 95.3% (v2 with FiLM conditioning) on Split-MNIST. However, probing reveals: 

• P5 (wrong context): ∆ = 0 .0pp — the model completely ignores context. 

• P6 (random θ0): accuracy drops catastrophically — all knowledge is in θ0.

• FiLM architecture contributes nothing (v1 ≈ v2 within noise). The root cause is the dimensionality mismatch: concatenating c ∈ R32 with θ ∈ R4842 creates a 150 :1

# D.2 Reproducibility 

Random seed control. All experiments use deterministic seeding via a shared utility: 

• torch.manual_seed(seed) 

• numpy.random.seed(seed) 

• random.seed(seed) 

• torch.cuda.manual_seed_all(seed) (when CUDA is available) Default seeds: {0, 1, 2, 3} for 4-seed experiments. All reported means and standard deviations are computed over these seeds unless otherwise noted. 

Result storage. Each experiment run produces a timestamped directory:

Configuration serialization. All hyperparameters are stored as JSON-serializable dictionaries in 

results.json , enabling exact reproduction. The configuration includes model architecture, optimizer settings, data preprocessing, and evaluation protocol.
